═══════════════════════════════════════════════════════════════════════

## 8. JAVASCRIPT MÍNIMO OBLIGATORIO
═══════════════════════════════════════════════════════════════════════

```javascript
(function(){
  const STORAGE_KEY = 'mooc_[nombre]_v2';
  const TOTAL = 15;

  function loadProgress(){ try{ return JSON.parse(localStorage.getItem(STORAGE_KEY))||{}; }catch(e){ return {}; } }
  function saveProgress(p){ localStorage.setItem(STORAGE_KEY, JSON.stringify(p)); }

  const progress = loadProgress();

  // Navegación
  document.getElementById('nav-list').addEventListener('click', e => {
    const item = e.target.closest('.nav-item');
    if(item && item.dataset.target){
      document.querySelectorAll('.module-section').forEach(s => s.classList.remove('active'));
      document.querySelectorAll('.nav-item').forEach(n => n.classList.remove('active'));
      document.getElementById(item.dataset.target).classList.add('active');
      item.classList.add('active');
    }
  });

  // Botones de completado
  document.addEventListener('click', e => {
    // Módulo completado
    const btn = e.target.closest('.complete-btn');
    if(btn && !btn.disabled){
      progress[btn.dataset.mod] = true;
      saveProgress(progress);
      updateUI();
    }
    // Quiz
    const opt = e.target.closest('.quiz-option');
    if(opt){
      const quiz = opt.closest('.quiz-block');
      if(quiz.dataset.answered) return;
      quiz.dataset.answered = '1';
      const correct = quiz.dataset.correct;
      const key = opt.dataset.key;
      quiz.querySelectorAll('.quiz-option').forEach(o => {
        if(o.dataset.key === correct) o.classList.add('correct');
        else if(o.dataset.key === key) o.classList.add('wrong');
      });
      const fb = quiz.querySelector('.quiz-feedback');
      fb.classList.add('show', key===correct ? 'ok' : 'fail');
      fb.textContent = key===correct ? '✓ Correcto' : '✗ Incorrecto — la correcta está en verde';
    }
  });

  function updateUI(){
    const done = Object.keys(progress).filter(k=>progress[k]).length;
    document.getElementById('done-count').textContent = done;
    document.getElementById('progress-fill').style.width = (done/TOTAL*100)+'%';
    document.querySelectorAll('.nav-item').forEach(item => {
      const mod = item.dataset.target?.replace('modulo-','');
      const check = item.querySelector('.nav-check');
      if(mod && progress[mod]){ item.classList.add('done'); if(check) check.textContent='✓'; }
    });
    document.querySelectorAll('.complete-btn').forEach(btn => {
      if(progress[btn.dataset.mod]){ btn.textContent='✓ Completado'; btn.classList.add('done'); btn.disabled=true; }
    });
  }

  updateUI();
})();
```

## 9. CONTENIDO POR MÓDULO: DENSIDAD Y CALIDAD
═══════════════════════════════════════════════════════════════════════

### A — Núcleo Teórico Pareto
- 3-5 subsecciones con h4 (no más)
- Al menos 1 tabla comparativa (spec)
- Al menos 1 bloque de código con ejemplo ejecutable
- Al menos 1 diagrama (diagram-box o SVG inline)
- Foco en el 20% del tema que da el 80% del valor práctico

### B — Laboratorio Guiado
- Objetivo concreto en 1 línea
- Input/output explícito (qué entra, qué sale)
- Pasos numerados con código
- Nota de rendimiento (tiempo/RAM estimado)

### C — Reto HERO por módulo
- Contexto realista de industria (empresa ficticia, proyecto real)
- MÍNIMO 3 restricciones técnicas con números concretos
- Entregable concreto (archivo, demo, informe)
- Métrica de éxito cuantitativa

### D — Drills
- MÍNIMO 5 ejercicios numerados (EX X.1 a EX X.5)
- Cada ejercicio con enunciado concreto y resultado esperado
- Mezcla de: cálculo, análisis, experimentación, comparación

### E — Espacio Feynman
- 1-2 preguntas collapsibles "¿Por qué...?" con respuesta profunda
- 1-2 quizzes de opción múltiple (data-correct="a|b|c|d")
- La respuesta correcta NO es la más obvia (evitar la A como default)

### F — Caja de Herramientas
- 4-8 herramientas relevantes para ese módulo específico
- Tipos: Doc, Lib, Paper, Tool, Plugin, Software, Dataset, YouTube
- Descripción en una línea que explica PARA QUÉ sirve en este contexto

   - Sidebar con los 15 nav-items
   - M0 COMPLETO (mayor densidad visual)
   - M14 COMPLETO (Hero final)
   - M1-M13 como PLACEHOLDERS: `<!-- CONTENT_MX -->`

2. **Paso 2**: Rellenar módulos por lotes de 2-3 con Edit tool:
   - M1+M2 en paralelo
   - M3+M4 en paralelo
   - M5+M6 en paralelo
   - M7+M8 en paralelo
   - M9+M10 en paralelo
   - M11+M12 en paralelo
   - M13 solo (cierre antes de M14)

3. **Verificación**: `grep -c "CONTENT_M" archivo.html` → debe dar 0

## 11. AUDITORÍA DE CONFORMIDAD (checklist por módulo)
═══════════════════════════════════════════════════════════════════════

Para verificar que un MOOC cumple la estructura, comprobar por módulo:
□ A: ¿Hay card con badge-theory "Núcleo Teórico Pareto" con tabla + código?
□ B: ¿Hay collapsible con badge-lab "Laboratorio Guiado" con código ejecutable?
□ C: ¿Hay collapsible con badge-hero "Reto HERO" con restricciones técnicas?
□ D: ¿Hay card con badge-drill y ≥5 ejercicios numerados EX X.N?
□ E: ¿Hay card/collapsible con badge-feynman y ≥1 quiz interactivo?
□ F: ¿Hay card con badge-tools y ≥4 tool-link con t-type, t-name, t-desc?

## 14. RESTRICCIONES DE CALIDAD
═══════════════════════════════════════════════════════════════════════
1. Prohibido explicar conceptos básicos de audio o electrónica. El usuario es experto.
2. Código siempre ejecutable: imports correctos, variables definidas, sin pseudocódigo vago.
3. Tablas comparativas siempre con ≥3 columnas y ≥3 filas de datos.
4. Rúbrica de M14 suma exactamente 100 puntos.
5. Quiz: la respuesta correcta tiene una justificación técnica en la answer card.
6. Herramientas en F siempre con URL o nombre suficiente para buscarla.
7. Sin comentarios vagos en código ("# hacer X aquí"). Solo comentarios informativos.
8. Los quizzes en E nunca tienen como correcta la primera opción (a) más de 2 veces en el mismo módulo.
