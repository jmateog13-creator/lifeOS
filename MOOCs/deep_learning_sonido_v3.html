M1 · CNN
CNN para Audio: ResNet, EfficientNet y AudioCNN propios
Receptive field, skip connections, kernels asimétricos, pooling estratégico

1.1 Receptive Field y ResNet

Cada capa expande lo que el output puede "ver" del input. RF_n = RF_{n-1} + (k-1) · stride_acumulado. Con 4 stages CNN → RF ≈ 100 frames temporales. A hop=512, sr=22050 → ~2.3s.

He et al. 2015. Skip connections: y = F(x) + x. Si F(x) → 0, la red puede ser identidad. Por defecto en cualquier audio CNN moderna.

import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, c_in, c_out, stride=1):
        super().__init__()
        self.conv1 = nn.Conv2d(c_in, c_out, 3, stride, 1, bias=False)
        self.bn1   = nn.BatchNorm2d(c_out)
        self.conv2 = nn.Conv2d(c_out, c_out, 3, 1, 1, bias=False)
        self.bn2   = nn.BatchNorm2d(c_out)
        self.skip  = nn.Identity() if (stride==1 and c_in==c_out) else \
                     nn.Conv2d(c_in, c_out, 1, stride, bias=False)
    def forward(self, x):
        h = torch.nn.functional.relu(self.bn1(self.conv1(x)))
        h = self.bn2(self.conv2(h))
        return torch.nn.functional.relu(h + self.skip(x))

LSTM cell y CRNN canónica

# LSTM cell (Hochreiter & Schmidhuber 1997)
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)    # forget gate
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)    # input gate
g_t = tanh(W_g · [h_{t-1}, x_t] + b_g) # candidate
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)    # output gate
C_t = f_t * C_{t-1} + i_t * g_t         # cell state (long-term)
h_t = o_t * tanh(C_t)                   # hidden state
import torch.nn as nn

class CRNN(nn.Module):
    def __init__(self, n_classes=10, hidden=256):
        super().__init__()
        self.cnn = nn.Sequential(
            nn.Conv2d(1, 32, 3, padding=1), nn.ReLU(), nn.MaxPool2d((2,1)),
            nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(), nn.MaxPool2d((2,1)),
            nn.Conv2d(64, 128, 3, padding=1), nn.ReLU(),
        )
        self.lstm = nn.LSTM(128*32, hidden, bidirectional=True, batch_first=True)
        self.fc   = nn.Linear(2*hidden, n_classes)

    def forward(self, x):   # (B, 1, n_mels=128, T)
        h = self.cnn(x)     # (B, 128, 32, T)
        B, C, F, T = h.shape
        h = h.permute(0, 3, 1, 2).reshape(B, T, C*F)
        h, _ = self.lstm(h) # (B, T, 2*hidden)
        return self.fc(h)   # (B, T, n_classes)

M3 · Generativos I
Autoencoders, VAE y β-VAE: comprimir y manipular
ELBO, reparameterization trick, posterior collapse, RAVE

VAE — distribución posterior parametrizada

Kingma & Welling 2014. El encoder devuelve μ(x) y log σ²(x). Sample: z = μ + σ·ε, ε~N(0,I) (reparameterization trick — diferenciable). Loss ELBO:

L = E[||x - g(z)||²] + β · KL(N(μ, σ²) || N(0, I))
import torch, torch.nn as nn

class MelVAE(nn.Module):
    def __init__(self, latent=128):
        super().__init__()
        self.enc = nn.Sequential(
            nn.Conv2d(1, 32, 4, 2, 1), nn.ReLU(),
            nn.Conv2d(32, 64, 4, 2, 1), nn.ReLU(),
            nn.Conv2d(64, 128, 4, 2, 1), nn.ReLU(), nn.Flatten(),
        )
        self.fc_mu     = nn.Linear(128*16*8, latent)
        self.fc_logvar = nn.Linear(128*16*8, latent)
        self.fc_dec    = nn.Linear(latent, 128*16*8)
        self.dec = nn.Sequential(
            nn.ConvTranspose2d(128, 64, 4, 2, 1), nn.ReLU(),
            nn.ConvTranspose2d(64, 32, 4, 2, 1), nn.ReLU(),
            nn.ConvTranspose2d(32, 1, 4, 2, 1),
        )

    def reparam(self, mu, logvar):
        return mu + (0.5*logvar).exp() * torch.randn_like(logvar)

    def forward(self, x):
        h = self.enc(x)
        mu, logvar = self.fc_mu(h), self.fc_logvar(h)
        z = self.reparam(mu, logvar)
        return self.dec(self.fc_dec(z).view(-1, 128, 16, 8)), mu, logvar

def vae_loss(x_rec, x, mu, logvar, beta=1.0):
    rec = torch.nn.functional.mse_loss(x_rec, x, reduction="sum")
    kl  = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return rec + beta * kl

β-VAE, posterior collapse y RAVE

β > 1: fuerza disentanglement (cada dimensión latente captura un factor puro — timbre, pitch). Posterior collapse: el decoder ignora z (KL→0). Solución: KL annealing (β arranca en 0, sube linealmente). RAVE (Caillon & Esling 2021): VAE waveform-domain + adversarial fine-tuning, inferencia <5ms en CPU, plugin nn~ para Max/MSP.

M4 · Generativos II
GANs para Audio: WGAN-GP, WaveGAN, MelGAN

WGAN-GP y Spectral Normalization

GAN vanilla: mode collapse, vanishing gradient cuando D es demasiado bueno. WGAN (Arjovsky 2017) reemplaza Jensen-Shannon por Earth Mover Distance. GP (Gulrajani 2017) penaliza el gradiente del crítico para forzar Lipschitz-1.

import torch
def gradient_penalty(D, real, fake, device):
    bsz = real.size(0)
    eps = torch.rand(bsz, 1, 1, 1, device=device)
    interp = (eps * real + (1 - eps) * fake).requires_grad_(True)
    d_interp = D(interp)
    grads = torch.autograd.grad(
        outputs=d_interp, inputs=interp,
        grad_outputs=torch.ones_like(d_interp),
        create_graph=True, retain_graph=True,
    )[0]
    grads = grads.view(bsz, -1)
    return ((grads.norm(2, dim=1) - 1) ** 2).mean()

# Loss D = E[D(fake)] - E[D(real)] + 10 * GP
# Loss G = -E[D(fake)]
# Ratio critico/G: 5:1

FAD — Fréchet Audio Distance

Distancia Fréchet entre distribuciones de embeddings VGGish de real vs generado. Métrica perceptual robusta. FAD <5 = "casi indistinguible". FAD >50 = "obvio". El crítico WGAN no usa sigmoid: estima Wasserstein (real-valued), no probabilidad.

Self-attention y AST

attn = softmax(Q · Kᵀ / sqrt(d_k)) · V

M9 · Streaming
Streaming y Procesado Causal: inferencia en tiempo real
Causal conv, chunk-wise processing, latency-controlled BiLSTM

Causalidad y Latencia

BiLSTM: necesita la secuencia completa → incompatible con streaming. Para tiempo real: LSTM unidireccional (causal por construcción), LC-BLSTM (backward limitado a ventana pequeña), Time-Restricted Self-Attention (atención solo a pasados N frames).

import torch.nn as nn

class CausalConv1d(nn.Module):
    """Conv1d causal: no mira el futuro."""
    def __init__(self, in_ch, out_ch, kernel_size, dilation=1):
        super().__init__()
        self.pad = (kernel_size - 1) * dilation
        self.conv = nn.Conv1d(in_ch, out_ch, kernel_size,
                              dilation=dilation, padding=0)
    def forward(self, x):
        x = nn.functional.pad(x, (self.pad, 0))
        return self.conv(x)

class StreamingLSTM(nn.Module):
    """LSTM con estado persistente entre chunks."""
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, batch_first=True)
        self.hidden = None

    def reset(self): self.hidden = None

    def forward(self, chunk):  # chunk: (B, T_chunk, F)
        out, self.hidden = self.lstm(chunk, self.hidden)
        return out
M10 · Quantization
Quantization: PTQ, QAT, INT8 en producción

PTQ vs QAT

Post-Training Quantization (PTQ): calibra el rango con un dataset de calibración; MinMax desperdicia rango por outliers. Quantization-Aware Training (QAT): la red aprende a evitar outliers de activación durante training simulado en INT8 → recupera más accuracy que PTQ. QAT es más costoso pero esencial para Transformers.

M11 · Robustez
Adversarial Robustness en Audio
FGSM, PGD, ataques físicos, adversarial training, randomized smoothing

Ataques y Defensas

FGSM (Goodfellow 2014): perturbación ε en dirección del gradiente. PGD (Madry 2018): multi-step FGSM con proyección → ataque más fuerte. Ataques físicos en audio: perturbaciones inaudibles que engañan ASR (Carlini & Wagner, CommanderSong).

import torch

def fgsm_attack(model, x, y, eps=0.01):
    """Fast Gradient Sign Method para audio."""
    x_adv = x.clone().requires_grad_(True)
    loss = torch.nn.functional.cross_entropy(model(x_adv), y)
    loss.backward()
    return (x_adv + eps * x_adv.grad.sign()).detach().clamp(-1, 1)

def pgd_attack(model, x, y, eps=0.01, alpha=0.002, n_steps=40):
    """Projected Gradient Descent."""
    x_adv = x.clone() + torch.empty_like(x).uniform_(-eps, eps)
    for _ in range(n_steps):
        x_adv.requires_grad_(True)
        loss = torch.nn.functional.cross_entropy(model(x_adv), y)
        loss.backward()
        x_adv = (x_adv + alpha * x_adv.grad.sign()).detach()
        x_adv = torch.max(torch.min(x_adv, x+eps), x-eps).clamp(-1, 1)
    return x_adv

Adversarial Training: incluir ejemplos adversariales en el training (AT usa AT accuracy alta vs ataques conocidos). Randomized Smoothing: certificación matemática pero baja accuracy clean. Elegir AT cuando se necesita accuracy práctica vs ataques conocidos; Smoothing para garantías formales.

Examen Feynman Final — Explica sin notas

Q1 · Adversarial Robustness

¿Cuándo usar Adversarial Training en lugar de Randomized Smoothing?

Ver respuesta

Adversarial Training cuando necesitas accuracy alta vs ataques conocidos y no requieres garantías matemáticas. Randomized Smoothing aporta certificación formal (garantía matemática del radio de perturbación tolerable) pero baja accuracy en datos limpios. AT prima accuracy práctica; Smoothing prima garantía formal. Para audio en producción (ASR, speaker ID), AT es más habitual por la mejor accuracy clean.

Q2 · Latent Diffusion

¿Por qué Latent Diffusion entrena más rápido que diffusion sobre waveform raw?

Ver respuesta

Porque la U-Net opera en el espacio latente comprimido por un VAE preentrenado (factor 8-64×). En lugar de trabajar con tensores de audio de longitud T=88k (4s @22kHz), opera en tensores de longitud T/64 ≈ 1.4k. Todas las T iteraciones de denoising procesan tensores mucho menores → entrenamiento 10-100× más rápido sin perder calidad perceptual significativa, ya que el VAE captura las estadísticas relevantes del audio.

Q3 · BiLSTM vs LSTM unidireccional

¿Cuándo elegir BiLSTM vs LSTM unidireccional para tareas de audio?

Ver respuesta

BiLSTM para tareas offline (transcripción de archivo, análisis post-hoc): capta contexto pasado y futuro → mejor accuracy. LSTM unidireccional para streaming/live: es causal por construcción, solo necesita los frames pasados → latencia mínima. Para ASR en producción live, LSTM unidireccional o LC-BLSTM (backward limitado a ventana pequeña) son la opción correcta aunque sacrifiquen algunos puntos de WER.

Q4 · QAT vs PTQ

¿Por qué QAT recupera más accuracy que PTQ en Transformers?

Ver respuesta

Porque QAT entrena la red para ser robusta a la cuantización INT8. Durante el training simulado con fake quantization, la red aprende a evitar outliers de activación que PTQ MinMax no puede suprimir. PTQ MinMax calibra el rango con datos de calibración, pero los outliers de activación en Transformers desperdician el rango dinámico INT8. QAT es más costoso (requiere fino 5-10 epochs) pero esencial para mantener accuracy en modelos grandes.

Q5 · Evaluación generativa

¿Qué reportar COMO MÍNIMO al comparar modelos generativos de audio?

Ver respuesta

FAD + listening test (MOS o ABX con ≥10 listeners) + diversidad (intra-distance entre muestras para detectar mode collapse) + análisis de modos. Nunca solo FAD: una métrica objetiva no capta todos los aspectos perceptuales. Incluir ejemplos cherry-picked Y ejemplos aleatorios (para no sesgar la presentación). Si hay speaker ID o voz: análisis de fairness por subgrupos demográficos. Reportar todo con intervalos de confianza y statistical significance (paired t-test, Cohen's d).
