M1 · Modelo ABT
El Modelo ABT de Randy Olson
And/But/Therefore: la estructura narrativa mínima viable para contenido educativo

A Núcleo Teórico

Randy Olson (biólogo marino convertido en cineasta) destila toda narrativa efectiva en tres conectores:

AND = contexto, acumulación. Muchos AND = aburrimiento.

BUT = conflicto, tensión. La chispa que activa la atención.

THEREFORE = consecuencia, resolución. Cierra el loop narrativo.

Ratio de narración = (# BUT / # AND) × 100 → óptimo: 10-29%
PosiciónEstructuraEjemploProblema
DHY (extremo no narrativo)Solo hechosPaper científicoSin tensión, olvidable
AAASolo ANDEnciclopediaAcumulación sin conflicto
ABTAND+BUT+THEREFOREPitch científicoEl punto óptimo
CliffhangerSolo BUTThriller sin finTensión sin resolución

B Laboratorio — Analizador ABT con Python

import re

def analyze_abt(transcript: str) -> dict:
    text_lower = transcript.lower()
    AND_WORDS  = ['y ', 'y,', 'además', 'también', 'asimismo', ' and ', 'moreover', 'furthermore']
    BUT_WORDS  = ['pero', 'sin embargo', 'aunque', 'no obstante', ' but ', 'however', 'yet']
    THERE_WORDS= ['por lo tanto', 'entonces', 'así que', 'por eso', 'therefore', 'thus', 'so ']

    def count_keywords(text, keywords):
        return sum(text.count(kw) for kw in keywords)

    n_and   = count_keywords(text_lower, AND_WORDS)
    n_but   = count_keywords(text_lower, BUT_WORDS)
    n_there = count_keywords(text_lower, THERE_WORDS)
    ratio_but  = (n_but  / max(n_and, 1)) * 100
    diagnosis  = (
        "ÓPTIMO: tensión narrativa equilibrada (10-29%)"
        if 10 <= ratio_but <= 29
        else "PLANO: demasiados AND, añadir conflicto" if ratio_but < 10
        else "CAÓTICO: demasiados BUT, reducir tensión"
    )
    return {'n_and': n_and, 'n_but': n_but, 'n_therefore': n_there,
            'ratio_but_and': f"{ratio_but:.1f}%", 'diagnosis': diagnosis}

def transform_to_abt(topic: str, pain_point: str, solution: str) -> str:
    return f"""[AND - CONTEXTO] {topic} es fundamental, y las herramientas existen.
[BUT - CONFLICTO] Pero {pain_point}, y esto bloquea al 90% de los estudiantes.
[THEREFORE - RESOLUCIÓN] Por lo tanto, {solution}."""

Q1 · Diagnóstico ABT

Un script tiene 15 conectores AND y 2 BUT. ¿Cuál es el ratio BUT/AND y qué diagnóstico corresponde según el modelo de Olson?

Ver respuesta

Ratio = (2/15) × 100 = 13.3%. Diagnóstico: ÓPTIMO. Está dentro del rango 10-29% de Randy Olson, lo que indica equilibrio correcto entre contexto (AND) y tensión narrativa (BUT). Con 13.3% hay suficiente conflicto para mantener la atención del espectador sin crear un exceso de tensión que resulte agotador.

M2 · Ecuación L=(c+a+b)×R×t
Ecuación de Aprendizaje
L = (c + a + b) × R × t — modelado cuantitativo del aprendizaje efectivo

A Núcleo Teórico

L = (c + a + b) × R × t
VariableSignificadoRangoCómo optimizar
c (claridad)Comprensión conceptual0-10Analogías, diagramas, Feynman technique
a (aplicabilidad)Usabilidad práctica inmediata0-10Laboratorios concretos, proyectos reales
b (belongingness)Conexión identitaria del aprendiz0-10Casos de uso del nicho del estudiante
R (retención)Tasa de retención a 7 días0-1Espaciado, retrieval practice, interleaving
t (tiempo activo)Tiempo de práctica deliberadahorasEliminar fricción, gamificación, streaks

Insight clave: L es multiplicativo, no aditivo. Si R=0.1 (sin repaso), el aprendizaje colapsa aunque c+a+b=30. La retención es el multiplicador más crítico.

B Laboratorio — Calculadora y simulador de Ebbinghaus

import numpy as np

def learning_score(c, a, b, R, t):
    return (c + a + b) * R * t

scenarios = {
    'MOOC pasivo (ver sin practicar)': {'c':8,'a':3,'b':5,'R':0.15,'t':10},
    'MOOC activo (labs + quizzes)':    {'c':8,'a':8,'b':6,'R':0.55,'t':12},
    'Bootcamp intensivo (inmersión)':  {'c':7,'a':9,'b':8,'R':0.40,'t':80},
    'Mentoría personalizada':          {'c':9,'a':9,'b':9,'R':0.70,'t':20},
}
for name, params in scenarios.items():
    print(f"{name:35s} L = {learning_score(**params):6.1f}")

def ebbinghaus_retention(t_hours, stability=24):
    return np.exp(-t_hours / stability)

# Análisis de sensibilidad: ¿qué variable impacta más en L?
base = {'c':7,'a':7,'b':6,'R':0.4,'t':20}
for var in ['c','a','b','R','t']:
    test = dict(base); test[var] *= 1.1
    delta_L = (learning_score(**test) - learning_score(**base)) / learning_score(**base) * 100
    print(f"+10% en {var} → ΔL = {delta_L:.1f}%")

Q2 · Multiplicatividad de L

En L=(c+a+b)×R×t, si duplicas el tiempo de estudio (t×2) pero reduces R a la mitad (sin repaso), ¿qué pasa con L y qué conclusión pedagógica se extrae?

Ver respuesta

L queda igual — t×2 cancela con R÷2, lo que muestra que estudiar más sin repasar no ayuda. La ecuación es multiplicativa: (c+a+b) × (R/2) × (2t) = (c+a+b) × R × t. Pedagógicamente, esto significa que doblar el tiempo de estudio sin implementar práctica espaciada no produce más aprendizaje que el original. La inversión debe ir en aumentar R (repaso espaciado, retrieval practice) más que en más horas de exposición pasiva.

La ventana de atención inicial determina si el espectador continúa o abandona

A Núcleo Teórico

En YouTube, la decisión de continuar o irse ocurre en los primeros 5 segundos. Los espectadores que superan el minuto 1 tienen 4× más probabilidad de llegar al final.

B Laboratorio — Analizador de hooks

import re
from dataclasses import dataclass, field
from typing import List

@dataclass
class HookAnalysis:
    text: str; score: float = 0.0
    techniques: List[str] = field(default_factory=list)
    feedback:   List[str] = field(default_factory=list)

def analyze_hook(text: str, max_seconds: int = 5, wpm: float = 150) -> HookAnalysis:
    words     = text.split()
    max_words = int(max_seconds * wpm / 60)
    score     = 0.0; techniques = []; feedback = []

    if len(words) <= max_words:
        score += 25; techniques.append(f"Longitud OK ({len(words)} palabras)")
    else:
        feedback.append(f"Demasiado largo: {len(words)} palabras (máx {max_words})")

    text_lower = text.lower()
    if '?' in text:
        score += 25; techniques.append("Pregunta (gap de curiosidad)")

    tension_words = ['pero', 'sin embargo', 'problema', 'error', 'falla', 'fail']
    if any(w in text_lower for w in tension_words):
        score += 20; techniques.append("Tensión narrativa detectada")
    else:
        feedback.append("Añade un elemento de tensión o conflicto")

    if re.search(r'\b\d+\b', text):
        score += 15; techniques.append("Número específico (credibilidad)")

    time_words = ['minutos', 'horas', 'días', 'semanas', 'minutes']
    if any(w in text_lower for w in time_words):
        score += 15; techniques.append("Promesa temporal")

    return HookAnalysis(text=text, score=score, techniques=techniques, feedback=feedback)

Q3 · Peor Hook Posible

¿Por qué empezar un video con "Hola! Hoy vamos a aprender sobre..." es el peor tipo de hook y qué debería reemplazarlo?

Ver respuesta

Porque no abre ningún gap de curiosidad ni tensión; confirma lo que el espectador ya sabe (hay un video sobre X), sin dar razón para seguir mirando en los 5 segundos críticos. El espectador ya vio el título y thumbnail — repetir esa información no aporta valor nuevo. En su lugar, debería comenzar directamente con el conflicto (BUT), una pregunta provocadora o in medias res con el resultado final del módulo para crear expectativa inmediata.

M4 · Sparklines Duarte
Sparklines — Nancy Duarte
La estructura de las grandes presentaciones: contraste entre lo que es y lo que podría ser

A Núcleo Teórico

Nancy Duarte analizó 200 años de grandes discursos. Descubrió que todos comparten la misma estructura de Sparkline: alternancia entre What IS (realidad actual) y What COULD BE (futuro posible).

Estructura Sparkline:

New Bliss (fin) → CALL TO ACTION → What IS → What COULD BE → What IS → What COULD BE... → What IS

Cada oscilación crea tensión y la resolución final (Call to Action) libera esa tensión dirigiendo al público hacia la acción.

Presentación icónicaPatrón Sparkline
MLK "I Have a Dream"IS: segregación → COULD BE: igualdad (×8 ciclos)
Steve Jobs iPhone 2007IS: teléfonos torpes → COULD BE: smartphone (×5 ciclos)
TED Talk promedioIS: problema → COULD BE: solución (×2-3 ciclos)

B Laboratorio — Generador de estructura Sparkline

from dataclasses import dataclass, field
from typing import List

@dataclass
class SparklineBeat:
    beat_type: str   # 'IS' | 'COULD_BE' | 'CTA' | 'OPENING'
    content:   str
    duration_s: int = 60

@dataclass
class SparklideStructure:
    title: str
    beats: List[SparklineBeat] = field(default_factory=list)

    def add_is(self, content: str, duration_s: int = 60):
        self.beats.append(SparklineBeat('IS', content, duration_s)); return self

    def add_could_be(self, content: str, duration_s: int = 60):
        self.beats.append(SparklineBeat('COULD_BE', content, duration_s)); return self

    def add_cta(self, content: str, duration_s: int = 30):
        self.beats.append(SparklineBeat('CTA', content, duration_s)); return self

    def tension_score(self) -> float:
        transitions = sum(1 for i in range(1, len(self.beats))
                          if self.beats[i].beat_type != self.beats[i-1].beat_type)
        return transitions / max(len(self.beats), 1)

talk = SparklideStructure("Por qué tu mezcla no suena profesional")
talk.add_is("Los productores novatos pasan semanas en una canción", 90)
talk.add_could_be("Con el flujo correcto, mezclas profesionalmente en 3h", 60)
talk.add_is("No existe una guía definitiva de EQ para bajo", 120)
talk.add_could_be("Un checklist que siempre funciona", 60)
talk.add_cta("Descarga el EQ Checklist y aplícalo en tu próxima mezcla", 60)
print(f"Tensión narrativa: {talk.tension_score():.2f}")

Q4 · New Bliss y Call to Action

¿Por qué la estructura Sparkline termina con "New Bliss" (COULD_BE) justo antes del Call to Action, en lugar de terminar con otro "What IS"?

Ver respuesta

Para que el público quede en un estado emocional positivo de posibilidad, justo antes del Call to Action; terminar en IS dejaría al público en el problema sin motivación para actuar. Nancy Duarte lo llama "New Bliss": el punto final donde el público ha experimentado múltiples oscilaciones IS↔COULD_BE y llega a un estado de máxima posibilidad percibida. El CTA en este momento convierte la emoción acumulada en acción concreta — el estado emocional positivo reduce la resistencia al cambio.

Q5 · Error de Posicionamiento del Instructor

Según la estructura Hero's Journey aplicada al edutainment, ¿cuál es el error más común en el diseño de cursos y cómo se corrige?

Ver respuesta

El instructor se posiciona como el héroe ("yo hice X, yo construí Y") en lugar de posicionar al estudiante como el protagonista del viaje. Hablar de los logros propios centra la narrativa en el instructor, desplazando al estudiante del rol protagónico. La corrección: "Tú vas a construir X. Yo te doy el mapa del territorio que tomé 5 años en aprender." El instructor es el mentor (Gandalf, Yoda, Dumbledore) que ha estado antes en ese camino y puede acelerar el viaje del héroe/estudiante.

import math

def script_calculator(words: int, wpm: float = 150, b_roll_ratio: float = 0.3) -> dict:
    narration_min  = words / wpm
    total_min      = narration_min / (1 - b_roll_ratio)

        'pattern_interrupts': f"{math.floor(total_min / 0.5):.0f} (cada 30s aprox.)",

for words in [300, 600, 1200, 2400]:

Q6 · Cálculo de Duración

Un script de 1800 palabras a 150 WPM con 30% de B-roll, ¿cuánto durará el video final y cómo se calcula?

Ver respuesta

~17 minutos. Cálculo: 1800/150 = 12 min de narración activa. El B-roll ocupa el 30% del tiempo total, por lo que la narración representa el 70%: total = 12 / 0.7 = 17.1 min. Interpretación: de 17 minutos totales, 12 minutos son de voz hablando y 5.1 minutos son de B-roll (secuencias de apoyo sin narración activa en ese instante).

B Laboratorio — Analizador de pacing por segmentos

from typing import List, Tuple

def analyze_pacing(transcript_with_timestamps: List[Tuple[float, str]]) -> dict:
    segments = {}; window_s = 30
    for ts, text in transcript_with_timestamps:
        bucket = int(ts // window_s) * window_s
        if bucket not in segments: segments[bucket] = []
        segments[bucket].append(text)

    results = []
    for start_s in sorted(segments.keys()):
        combined = ' '.join(segments[start_s])
        words    = len(combined.split())
        wpm      = words / (window_s / 60)
        status   = ("OPTIMO" if 130 <= wpm <= 160
                    else "LENTO" if wpm < 130 else "RAPIDO")
        results.append({'time': f"{start_s//60:.0f}m{start_s%60:02.0f}s",
                        'wpm': round(wpm, 1), 'status': status})

    avg_wpm = sum(r['wpm'] for r in results) / len(results) if results else 0
    return {'segments': results, 'average_wpm': avg_wpm}

Q7 · Pausa Dramática y Neurología

¿Por qué una pausa de 0.5s antes de revelar la respuesta mejora la retención del espectador desde el punto de vista neurológico?

Ver respuesta

Activa el efecto de anticipación: el cerebro genera predicciones activas durante la pausa, creando un estado de alta atención; la revelación posterior tiene mayor impacto neurológico (dopamine release). Durante la pausa, el sistema predictivo del cerebro formula hipótesis sobre la respuesta — este estado de predicción activa el sistema de recompensa dopaminérgico. Cuando llega la revelación, independientemente de si la predicción fue correcta o no, hay un pico de activación que consolida la memoria del contenido.

B Laboratorio — Planificador automático de pattern interrupts

from dataclasses import dataclass, field
from typing import List
import random

@dataclass
class PatternInterrupt:
    time_s: float; type: str; description: str; duration_s: float = 2.0

@dataclass
class VideoInterruptPlan:
    title: str; duration_s: float
    interrupts: List[PatternInterrupt] = field(default_factory=list)

    def auto_plan(self, interval_s: float = 25, variety: bool = True):
        INTERRUPT_TYPES = [
            ('jump_cut',     'Jump cut + zoom in', 0.5),
            ('broll',        'B-roll ilustrativo', 5.0),
            ('graphic',      'Gráfico animado', 3.0),
            ('sfx',          'Sound effect de énfasis', 0.5),
            ('text_overlay', 'Texto clave en pantalla', 2.0),
            ('question',     'Pregunta retórica + pausa', 4.0),
        ]
        t = interval_s; i = 0
        while t < self.duration_s - 10:
            itype, desc, dur = INTERRUPT_TYPES[i % len(INTERRUPT_TYPES)]
            self.interrupts.append(PatternInterrupt(t, itype, desc, dur))
            t += interval_s + random.uniform(-5, 5); i += 1
        return self

    def validate(self):
        times  = sorted(pi.time_s for pi in self.interrupts)
        prev   = 0; issues = []
        for t in times:
            if t - prev > 30: issues.append(f"Gap de {t-prev:.0f}s entre {prev:.0f}s y {t:.0f}s")
            prev = t
        return issues if issues else ["Plan válido: sin gaps > 30s"]

random.seed(42)
plan = VideoInterruptPlan("M2 - MFCC Pipeline", duration_s=8*60)
plan.auto_plan(interval_s=22)
print("Interrupciones planificadas:", len(plan.interrupts))
print("Validación:", plan.validate())

Q8 · Orienting Response y Edutainment

¿Por qué el orienting response (respuesta de orientación) es relevante para el edutainment y cómo lo explotan intencionalmente los pattern interrupts?

Ver respuesta

Es el reflejo evolutivo que dirige la atención hacia cambios en el entorno (nuevos estímulos). Los pattern interrupts lo activan intencionalmente, manteniendo al cerebro en estado de alerta y procesamiento activo durante el video. Evolutivamente, prestar atención a cambios en el entorno es una ventaja de supervivencia — el sistema nervioso está cableado para responder a la novedad. Los creadores de edutainment explotan este mecanismo al introducir cambios visuales, sonoros o narrativos cada 20-30 segundos, previniendo que el espectador entre en modo piloto automático.

B Laboratorio — Procesador de voz tutorial con Python

import numpy as np, librosa, soundfile as sf
from scipy.signal import butter, sosfilt

def process_tutorial_voice(input_path: str, output_path: str, sr_target: int = 44100):
    y, sr = librosa.load(input_path, sr=sr_target, mono=True)

    # 1. High-pass filter (80 Hz)
    sos = butter(4, 80, btype='high', fs=sr, output='sos')
    y   = sosfilt(sos, y)

    # 2. EQ paramétrico simplificado
    def parametric_eq(audio, freq, gain_db, Q, fs):
        A   = 10 ** (gain_db / 40); w0 = 2 * np.pi * freq / fs
        alpha = np.sin(w0) / (2 * Q)
        b = [1 + alpha*A, -2*np.cos(w0), 1 - alpha*A]
        a = [1 + alpha/A, -2*np.cos(w0), 1 - alpha/A]
        from scipy.signal import lfilter
        return lfilter(b, a, audio)

    y = parametric_eq(y, freq=250,  gain_db=-3, Q=0.7,  fs=sr)  # scoop mud
    y = parametric_eq(y, freq=3000, gain_db=+2, Q=1.5,  fs=sr)  # presencia

    # 3. Normalizar a -3 dBFS
    peak   = np.abs(y).max(); target = 10 ** (-3 / 20)
    y      = y * (target / peak)

    sf.write(output_path, y, sr)
    print(f"Peak final: {20*np.log10(np.abs(y).max()):.1f} dBFS")
    print(f"RMS final:  {20*np.log10(np.sqrt(np.mean(y**2))):.1f} dBFS")

Q9 · High-Pass Filter en Voz Tutorial

¿Por qué se aplica un high-pass filter a 80 Hz en el procesamiento de voz tutorial y qué ocurre con el headroom si se omite?

Ver respuesta

Para eliminar el ruido de baja frecuencia (rumble de HVAC, vibraciones de mesa, manipulación de micrófono) que no contiene información de voz pero ocupa headroom y enturbia el mix. Si se omite, las bajas frecuencias parásitas consumen amplitud en el limitador/compresor final, reduciendo el headroom disponible para la voz. Esto obliga a bajar el nivel general o resulta en clipping. Las frecuencias bajo 80 Hz no contribuyen a la inteligibilidad de la voz humana, por lo que su eliminación no afecta la calidad perceptual.

import numpy as np, pandas as pd
from typing import List, Dict

def analyze_retention_curve(retention_data: List[Dict]) -> Dict:
    df = pd.DataFrame(retention_data).sort_values('time_pct')
    df['drop'] = df['retention_pct'].diff().fillna(0)
    drops = df[df['drop'] < -5].to_dict('records')
    avd_pct         = float(df.set_index('time_pct')['retention_pct'].mean())
    retention_30s   = float(df[df['time_pct'] <= 10]['retention_pct'].mean())
    completion_rate = float(df[df['time_pct'] >= 95]['retention_pct'].mean())
    return {
        'avd_pct':         round(avd_pct, 1),
        'retention_30s':   round(retention_30s, 1),
        'completion_rate': round(completion_rate, 1),
        'n_drops':         len(drops),
        'drop_timestamps': [f"{d['time_pct']:.0f}%" for d in drops[:3]],
    }

def benchmark_assessment(kpis: Dict) -> List[str]:
    recs = []
    if kpis['retention_30s'] < 70:
        recs.append("Hook débil: reforzar los primeros 5s con pregunta o tensión")
    if kpis['avd_pct'] < 40:
        recs.append("Edición lenta: aumentar pacing, más pattern interrupts")
    if kpis['completion_rate'] < 30:
        recs.append("Cierre débil: añadir teaser del próximo módulo en los últimos 30s")
    if not recs: recs.append("Métricas por encima de benchmark en todos los KPIs")
    return recs

Q10 · CTR Alto / AVD Bajo

Un video tiene CTR=8% (excelente) pero AVD=22% (muy bajo). ¿Cuál es el diagnóstico más probable y qué produce esta combinación?

Ver respuesta

El thumbnail/título atrae clics pero el contenido no cumple la promesa (clickbait); hay desconexión entre expectativa y entrega — los espectadores se van rápido. Esta combinación es la señal más clara de "bait-and-switch" narrativo: el espectador hace clic porque el título/thumbnail promete algo concreto, pero el contenido real no entrega esa promesa en los primeros minutos. El algoritmo de YouTube pesa el AVD más que el CTR para distribución, por lo que un alto CTR con bajo AVD resulta en penalización algorítmica progresiva.

Q11 · Leaderboards y Self-Determination Theory

¿Por qué los leaderboards globales pueden ser contraproducentes en cursos educativos, según la Self-Determination Theory?

Ver respuesta

Desplazan la motivación intrínseca (curiosidad, competencia real) por motivación extrínseca (ranking); los estudiantes en posiciones bajas pierden autonomía percibida y abandonan, lo que daña la completion rate. La SDT establece que la motivación intrínseca (aprender porque es interesante) es más sostenible y produce aprendizaje más profundo. Un leaderboard global coloca a la mayoría de usuarios en posiciones "perdedoras", dañando la percepción de competencia — una de las tres necesidades básicas de la SDT — y aumentando el abandono entre los estudiantes más rezagados, que son precisamente los que más necesitan apoyo.

Q12 · Restricción Temporal y Calidad Narrativa

¿Por qué el método South Park de "6 días" produce narrativas más limpias que un proceso sin deadline en términos del modelo ABT?

Ver respuesta

La restricción temporal fuerza la síntesis: solo queda lo esencial del AND-BUT-THEREFORE; sin deadline, el creador rellena con contexto innecesario (muchos AND) que diluye el conflicto central. La abundancia de tiempo disponible permite — y psicológicamente invita a — añadir más contexto, ejemplos y matices (más AND), lo que reduce el ratio BUT/AND. La restricción de 6 días obliga a priorizar: si solo hay tiempo para 3 frases, tienen que ser las 3 frases del ABT. La escasez elimina lo no esencial, produciendo el ratio óptimo de Olson (10-29%) de forma natural.

import json
from pathlib import Path

def transcribe_to_srt(audio_path: str, output_srt: str,
                       model_size: str = 'medium', language: str = 'es') -> str:
    import whisper
    model  = whisper.load_model(model_size)
    result = whisper.transcribe(model, audio_path, language=language,
                                 word_timestamps=True, verbose=False)
    srt_lines = []
    for i, seg in enumerate(result['segments'], 1):
        start = format_srt_time(seg['start'])
        end   = format_srt_time(seg['end'])
        srt_lines.extend([str(i), f"{start} --> {end}", seg['text'].strip(), ""])
    srt_content = '\n'.join(srt_lines)
    Path(output_srt).write_text(srt_content, encoding='utf-8')
    return srt_content

def format_srt_time(seconds: float) -> str:
    h = int(seconds // 3600); m = int((seconds % 3600) // 60)
    s = int(seconds % 60);    ms = int((seconds - int(seconds)) * 1000)
    return f"{h:02d}:{m:02d}:{s:02d},{ms:03d}"

def generate_distribution_plan(video_title: str, duration_min: float) -> dict:
    is_short = duration_min <= 3; is_medium = 3 < duration_min <= 20
    platforms = {
        'YouTube Main':       {'suitable': True,      'checklist': ['Thumbnail A/B', 'Descripción con timestamps', 'Subtítulos SRT revisados', 'Cards y End Screen']},
        'YouTube Shorts':     {'suitable': is_short,  'checklist': ['Corte vertical del hook', 'Texto overlay ≥24pt', '#Shorts en descripción']},
        'LinkedIn':           {'suitable': is_medium, 'checklist': ['Subtítulos burned-in (85% muted)', 'Primer frame informativo']},
    }
    return platforms

Q1 · ABT y Ratio

Un script de 400 palabras tiene 8 conectores AND y 1 BUT. ¿Cuál es el ratio BUT/AND, está en rango óptimo, y necesitas añadir más BUT para mejorar la narrativa?

Ver respuesta

Ratio = 1/8 × 100 = 12.5% — ya está en rango óptimo (10-29%). No es necesario añadir más BUT. El script ya tiene tensión narrativa adecuada según el modelo de Randy Olson. El rango 10-29% significa que hay suficiente conflicto para activar la atención sin crear un exceso de tensión que resulte agotador o incoherente. Con 12.5%, el script está hacia el extremo inferior del rango óptimo, lo que lo hace apropiado para contenido educativo (que requiere más contexto AND que contenido de entretenimiento puro).

Q2 · Ecuación de Aprendizaje Comparativa

Curso A: c=8, a=9, b=7, R=0.2, t=40h. Curso B: c=7, a=7, b=6, R=0.6, t=20h. ¿Cuál tiene mayor L y qué variable explica el resultado?

Ver respuesta

Curso A: (8+9+7)×0.2×40 = 24×8 = 192. Curso B: (7+7+6)×0.6×20 = 20×12 = 240. El Curso B produce más aprendizaje con la mitad del tiempo, porque la retención (R=0.6 vs 0.2) es el multiplicador más poderoso. R=0.6 es 3 veces mayor que R=0.2, mientras que el Curso A solo compensa con variables lineales (c+a+b=24 vs 20, y t=40 vs 20). La lección: invertir en práctica espaciada y retrieval practice (aumentar R) tiene mayor retorno que añadir más horas de exposición pasiva.

Q3 · Sparkline y Posición del CTA

¿En qué punto exacto del Sparkline va el "Call to Action" y por qué ese momento específico maximiza la conversión?

Ver respuesta

Al final, después del último COULD BE (New Bliss): el público está en el estado emocional más alto, con máxima motivación y mínima resistencia al cambio. La estructura Sparkline construye emoción acumulativa mediante múltiples oscilaciones IS↔COULD_BE. Al llegar al CTA, el público ha experimentado repetidamente el contraste entre la realidad actual (frustración) y el futuro posible (aspiración). En el pico emocional del New Bliss, la resistencia cognitiva al cambio está en su mínimo — el CTA en ese momento convierte la energía emocional acumulada en acción concreta con máxima eficiencia.

Q4 · Instructor como Mentor vs Héroe

Un instructor de curso online dice: "Yo construí este sistema para Netflix y en este curso te enseño mi método." ¿Qué error narrativo comete y cómo se reformula?

Ver respuesta

Se posiciona como el héroe en lugar del mentor: el instructor es Gandalf/Yoda, no Frodo/Luke. Hablar de sus logros propios centra la narrativa en él, desplazando al estudiante del viaje. La reformulación correcta: "Tú vas a construir un sistema de recomendación. Yo te doy el mapa del territorio que tomé 5 años en aprender — para que tú lo hagas en 6 semanas." Esta versión: (1) posiciona al estudiante como protagonista ("Tú vas a construir"), (2) convierte la experiencia del instructor en una herramienta al servicio del estudiante, (3) crea expectativa concreta y alcanzable.

Q5 · Diagnóstico de Analytics

Tu video tiene Retention at 30s = 85% pero AVD = 18%. ¿Qué parte de tu producción está fallando y cuáles son las primeras 3 acciones de mejora?

Ver respuesta

El cuerpo del video: el hook es excelente (85% supera el minuto 0.5) pero el contenido posterior pierde al espectador rápidamente. El pacing, la densidad técnica o la falta de pattern interrupts causan la caída. Primeras 3 acciones: (1) Revisar la curva de retención en YouTube Studio para identificar los timestamps exactos de mayor drop — ahí está el problema específico. (2) Añadir pattern interrupts cada 20-25s en las secciones con mayor caída (zoom cuts, B-roll, gráficos). (3) Simplificar las secciones técnicas más densas o dividir el video en episodios más cortos — el AVD de 18% en un video de duración media indica que el espectador abandona muy temprano, posiblemente en el minuto 1-2.
