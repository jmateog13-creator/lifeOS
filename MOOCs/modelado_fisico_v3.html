M1 · Ecuación de Onda
PDE de Onda 1D y Solución de D'Alembert
La base de toda síntesis por waveguide: dos ondas viajeras

PDE y D'Alembert

∂²y/∂t² = c² · ∂²y/∂x² donde c = √(K/ε) (velocidad de onda)
y(x,t) = y⁺(t − x/c) + y⁻(t + x/c) (solución de D'Alembert)

y⁺ viaja hacia la derecha; y⁻ hacia la izquierda. Esta dicotomía es exactamente la arquitectura de las Digital Waveguides: dos líneas de retardo.

f₀ = c/(2L) = (1/2L) · √(K/ε) (cuerda fijada en ambos extremos)
import numpy as np

def simulate_wave_equation(L=1.0, c=343.0, T=0.02, dx=0.01):
    dt = 0.9 * dx / c   # Criterio CFL: dt <= dx/c
    Nx = int(L / dx) + 1
    Nt = int(T / dt)
    x = np.linspace(0, L, Nx)
    y = np.zeros((Nt, Nx))
    pluck_pos = int(0.3 * Nx)
    y[0, :pluck_pos] = np.linspace(0, 0.01, pluck_pos)
    y[0, pluck_pos:] = np.linspace(0.01, 0, Nx - pluck_pos)
    y[1] = y[0].copy()
    r = (c * dt / dx) ** 2
    print(f"CFL ratio r = {r:.4f} (debe ser <= 1 para estabilidad)")
    for n in range(1, Nt - 1):
        y[n+1, 1:-1] = (2 * y[n, 1:-1] - y[n-1, 1:-1] +
                         r * (y[n, 2:] - 2*y[n, 1:-1] + y[n, :-2]))
        y[n+1, 0] = 0.0
        y[n+1, -1] = 0.0
    return y

print(f"f₀ teórica: {343.0/(2*0.39):.1f} Hz")
M2 · Digital Waveguides
Impedancia de Onda, Uniones de Dispersión y Red Acústica
Buffers circulares bidireccionales, scattering junction, reflexión y transmisión

Impedancia Rw y Scattering Junction

Rw ≜ F⁺/V⁺ = √(Kε)
k = (Rw₂ − Rw₁)/(Rw₂ + Rw₁) (coeficiente de reflexión)
import numpy as np

class DigitalWaveguide:
    def __init__(self, freq, fs=44100, loop_gain=0.996):
        self.fs = fs
        self.N = int(fs / freq)
        self.loop_gain = loop_gain
        self.right_going = np.zeros(self.N)  # y⁺
        self.left_going  = np.zeros(self.N)  # y⁻
        self.ptr = 0
        print(f"DWG: f={freq}Hz → L={self.N} samples")

    def excite(self, excitation):
        n = min(len(excitation), self.N)
        self.right_going[:n] = excitation[:n] / 2
        self.left_going[:n]  = excitation[:n] / 2

    def process_sample(self):
        right_end = self.right_going[(self.ptr + self.N - 1) % self.N]
        reflected  = -self.loop_gain * right_end
        self.left_going[self.ptr] = reflected
        left_end   = self.left_going[(self.ptr + 1) % self.N]
        self.right_going[(self.ptr + 1) % self.N] = -self.loop_gain * left_end
        output = self.right_going[self.ptr] + self.left_going[self.ptr]
        self.ptr = (self.ptr + 1) % self.N
        return output

    def synthesize(self, duration_s):
        n = int(duration_s * self.fs)
        self.excite(np.random.randn(self.N))
        return np.array([self.process_sample() for _ in range(n)])

# Sintetizar La4 (440 Hz)
dw = DigitalWaveguide(freq=440, loop_gain=0.9995)
output = dw.synthesize(duration_s=2.0)
M3 · Masa-Resorte
Sistemas MSD y el Método de Euler Simpléctico
Percusiones y golpes — integración simpléctica para preservar energía

Segunda Ley de Newton y Euler Simpléctico

m·x''(t) + Z·x'(t) + k·x(t) = F_ext(t)
v_{n+1} = v_n + h · (F_n / m) (actualizar velocidad primero)
x_{n+1} = x_n + h · v_{n+1} (usar velocidad YA actualizada)

El Euler Explícito inyecta energía artificialmente (inestable para ω₀·h grande). El Simpléctico preserva el área en el espacio de fase (x,v): la elipse de energía constante se mantiene como elipse sin expandirse.

import numpy as np
from scipy.io import wavfile

def msd_euler_symplectic(m, Z, k, F_ext, h, n_steps):
    """Euler Simpléctico: estable, preserva estructura simpléctica."""
    x, v = 0.0, 0.0
    positions = np.zeros(n_steps)
    for n in range(n_steps):
        F = F_ext[n] if n < len(F_ext) else 0.0
        a = (F - Z*v - k*x) / m
        v = v + h * a   # Primero v (actualizado)
        x = x + h * v   # Luego x con v ya actualizado
        positions[n] = x
    return positions

# Parámetros: tímpano (m=0.01kg, k=100 N/m, Z=0.5 Ns/m)
fs = 44100; h = 1.0 / fs
m, Z, k = 0.01, 0.5, 100.0
omega_0 = np.sqrt(k/m)
print(f"Frecuencia resonancia: {omega_0/(2*np.pi):.1f} Hz")
print(f"ω₀·h = {omega_0*h:.6f} (debe ser << 1)")

M4 · Karplus-Strong
El Algoritmo Karplus-Strong: Buffer Circular + Filtro Promedio
El caso más simple de DWG — mínimo cómputo, timbre de cuerda creíble

Ecuación de Diferencias KS

y(n) = b · 0.5 · [y(n−L) + y(n−L−1)] (b ≈ 0.996 para sustain largo)
f = Fs / L (L = longitud en samples del buffer)

El filtro de promediado actúa como pasa-bajos: las frecuencias altas decaen más rápido que las bajas, exactamente como en una guitarra real. Con solo 2 operaciones aritméticas por muestra, se obtiene un timbre de cuerda creíble.

import numpy as np
from scipy.io import wavfile

def karplus_strong(freq, duration_s, fs=44100, feedback=0.996):
    L = int(fs / freq)
    print(f"KS: L={L} samples, f={fs/L:.2f} Hz (objetivo: {freq} Hz)")
    n_samples = int(duration_s * fs)
    buffer = np.random.randn(L)
    output = np.zeros(n_samples)
    ptr = 0
    for n in range(n_samples):
        current = buffer[ptr]
        next_ptr = (ptr + 1) % L
        avg = feedback * 0.5 * (current + buffer[next_ptr])
        buffer[ptr] = avg
        output[n] = current
        ptr = (ptr + 1) % L
    return output

A_major = [220.0, 277.18, 329.63]
chord = sum(karplus_strong(f, 4.0, feedback=0.998) for f in A_major) / len(A_major)
chord /= np.max(np.abs(chord)) + 1e-8

M5 · Síntesis Conmutada
Commuted Synthesis: Pre-convolucionando el Cuerpo
Axioma LTI: el IR del cuerpo se pre-convoluciona con la excitación

Axioma de Conmutación LTI

Salida = Excitación * Cuerda * Cuerpo = (Excitación * Cuerpo) * Cuerda
Buffer_inicial = noise * h_cuerpo(t) (pre-cómputo offline)

En tiempo real, solo se procesa la waveguide: el cuerpo está "cocinado" en la excitación inicial. Reducción drástica del costo computacional. No funciona cuando el cuerpo es no-lineal (viola la condición LTI).

import numpy as np
from scipy.signal import fftconvolve
from scipy.io import wavfile

def create_body_ir(resonant_freqs, amplitudes, decay_times, fs=44100, duration=0.5):
    n = int(duration * fs)
    t = np.arange(n) / fs
    ir = np.zeros(n)
    for f, a, tau in zip(resonant_freqs, amplitudes, decay_times):
        decay_rate = np.log(1000) / (tau / 1000)
        ir += a * np.exp(-decay_rate * t) * np.sin(2 * np.pi * f * t)
    return ir / (np.max(np.abs(ir)) + 1e-8)

def commuted_synthesis_guitar(freq, duration_s, fs=44100, feedback=0.997):
    # Resonancias típicas de cuerpo de guitarra
    body_ir = create_body_ir(
        [80, 100, 140, 200, 280, 350],
        [1.0, 0.8, 0.6, 0.5, 0.3, 0.2],
        [500, 400, 300, 200, 150, 100], fs
    )
    L = int(fs / freq)
    raw_noise = np.random.randn(L)
    body_noise = fftconvolve(raw_noise, body_ir, mode='same')[:L]
    body_noise /= np.max(np.abs(body_noise)) + 1e-8

    buffer = body_noise.copy()
    n_samples = int(duration_s * fs)
    output = np.zeros(n_samples)
    ptr = 0
    for n in range(n_samples):
        current = buffer[ptr]
        next_ptr = (ptr + 1) % L
        buffer[ptr] = feedback * 0.5 * (current + buffer[next_ptr])
        output[n] = current
        ptr = (ptr + 1) % L
    return output
M6 · Loop Filter y EDR
Identificación y Diseño del Loop Filter desde Grabaciones Reales
STFT, Energy Decay Relief, regresión, invfreqz

EDR y Pipeline de Identificación

EDR(t_k, ω_m) = EDR(t_{k+1}, ω_m) + |X(t_k, ω_m)|²
|H_loop(ω)| = e^{−τ(ω)·π/fs} donde τ(ω) es la tasa de decaimiento medida

El pipeline: (1) F0 extraction → (2) STFT con ventana Hamming de ~4 períodos → (3) EDR 3D → (4) regresión lineal para tasa de decaimiento por frecuencia → (5) invfreqz para diseñar el IIR.

import numpy as np
from scipy.signal import stft
from scipy.stats import linregress

def compute_edr(audio, fs, hop_size=256, fft_size=4096):
    freqs, times, Zxx = stft(audio, fs=fs, nperseg=fft_size,
                              noverlap=fft_size-hop_size, window='hamming')
    energy = np.abs(Zxx) ** 2
    edr = np.cumsum(energy[:, ::-1], axis=1)[:, ::-1]
    edr_db = 10 * np.log10(edr + 1e-12)
    edr_db -= edr_db[:, 0:1]
    return edr_db, freqs, times

def extract_decay_rates(edr_db, freqs, times):
    decay_rates = {}
    for bin_idx in range(0, len(freqs), 10):
        f = freqs[bin_idx]
        decay_curve = edr_db[bin_idx, :]
        mask = (decay_curve <= 0) & (decay_curve >= -30)
        if np.sum(mask) < 5: continue
        slope, _, _, _, _ = linregress(times[mask], decay_curve[mask])
        decay_rates[f] = slope
    return decay_rates

# Señal simulada (E2 de guitarra: fundamental + 2 armónicos)
fs = 44100; t = np.arange(3 * fs) / fs; f0 = 82.4
audio_sim = (np.exp(-0.5*t) * np.sin(2*np.pi*f0*t) +
             0.5*np.exp(-1.0*t) * np.sin(2*np.pi*2*f0*t) +
             0.25*np.exp(-2.0*t) * np.sin(2*np.pi*3*f0*t))

edr, freqs, times = compute_edr(audio_sim, fs)
decay_rates = extract_decay_rates(edr, freqs)
for f, rate in list(decay_rates.items())[:4]:
    rt60 = -60 / rate if rate < 0 else float('inf')
    print(f"  f={f:.1f}Hz: {rate:.1f}dB/s (RT60~{rt60:.2f}s)")
M7 · Retraso Fraccionario
Interpolación Sub-Muestra para Afinación Precisa
JND ±0.5 cent, interpolación Lagrange, filtro allpass

JND y Métodos de Interpolación

Para 440 Hz a 44.100 Hz: L = 100.227 samples. Con L=100 (entero) → frecuencia real = 441 Hz (+3.9 cents): perceptiblemente desafinado (JND ≈ ±0.5 cent). Se necesita retraso fraccionario.

y(n−η) = (1−η)·y(n) + η·y(n−1) (lineal — introduce Frequency Warping)
H_allpass(z) = (a + z⁻¹)/(1 + a·z⁻¹) a = (1−η)/(1+η)
import numpy as np
from scipy.io import wavfile

class FractionalDelayKS:
    def __init__(self, freq, fs=44100, feedback=0.997, interp_order=4):
        self.fs = fs; self.feedback = feedback
        total_delay = fs / freq
        self.L = int(total_delay)
        self.frac = total_delay - self.L
        self.actual_freq = fs / (self.L + self.frac)
        print(f"KS Fraccionario: error = {abs(self.actual_freq-freq)/freq*1200:.4f} cents")
        self.buffer = np.zeros(self.L + interp_order + 4)
        self.ptr = 0; self.order = interp_order
        self.lagrange_coeffs = self._lagrange_coeffs(self.frac, interp_order)

    def _lagrange_coeffs(self, frac, N):
        coeffs = np.zeros(N + 1)
        for k in range(N + 1):
            prod = 1.0
            for m in range(N + 1):
                if m != k: prod *= (frac - m) / (k - m)
            coeffs[k] = prod
        return coeffs

    def _read_fractional(self):
        return sum(c * self.buffer[(self.ptr + self.L - k) % len(self.buffer)]
                   for k, c in enumerate(self.lagrange_coeffs))

    def synthesize(self, duration_s):
        self.buffer[:self.L] = np.random.randn(self.L)
        n = int(duration_s * self.fs)
        output = np.zeros(n)
        for i in range(n):
            delayed = self._read_fractional()
            prev_idx = (self.ptr + self.L - 1) % len(self.buffer)
            self.buffer[self.ptr] = self.feedback * 0.5 * (delayed + self.buffer[prev_idx])
            output[i] = delayed
            self.ptr = (self.ptr + 1) % len(self.buffer)
        return output

ks_frac = FractionalDelayKS(440.0, fs=44100, feedback=0.998)
output = ks_frac.synthesize(3.0)
output /= np.max(np.abs(output)) + 1e-8

M9 · JUCE DelayLine
Physical Modeling en JUCE: dsp::DelayLine y ProcessorChain
Karplus-Strong completo como plugin VST3/AU en C++

Arquitectura JUCE DSP para Waveguides

MIDI → Excitación → [DelayLine + LoopFilter + Gain] → Output
// JUCE KS Voice — fragmento clave
#pragma once
#include <JuceHeader.h>

class KarplusStrongVoice : public juce::SynthesiserVoice {
public:
    KarplusStrongVoice() {
        delayLine.setMaximumDelayInSamples(4096);
        auto coeffs = juce::dsp::IIR::Coefficients<float>::makeLowPass(44100, 8000.0f);
        *loopFilter.state = *coeffs;
    }

    void startNote(int midiNote, float velocity, juce::SynthesiserSound*, int) override {
        const double fs = getSampleRate();
        const double freq = juce::MidiMessage::getMidiNoteInHertz(midiNote);
        delayLength = fs / freq;  // Retraso fraccionario Lagrange3rd
        loopGain = 0.9995f * velocity;
        juce::Random rng;
        for (int i = 0; i < (int)delayLength; ++i)
            delayLine.pushSample(0, rng.nextFloat() * 2.0f - 1.0f);
        isActive = true;
    }

    void renderNextBlock(juce::AudioBuffer<float>& buf, int start, int num) override {
        if (!isActive) return;
        for (int n = start; n < start + num; ++n) {
            float delayed  = delayLine.popSample(0, delayLength, true);
            float filtered = loopFilter.processSample(delayed);
            delayLine.pushSample(0, loopGain * filtered);
            buf.addSample(0, n, delayed);
            buf.addSample(1, n, delayed);
        }
    }

private:
    juce::dsp::DelayLine<float, juce::dsp::DelayLineInterpolationTypes::Lagrange3rd> delayLine;
    juce::dsp::IIR::Filter<float> loopFilter;
    float delayLength = 100.0f, loopGain = 0.996f;
    bool isActive = false;
};

dsp::ProcessorChain usa template metaprogramming (variadic templates): los procesadores se resuelven en compilación generando código inline sin vtable dispatch → 2-5× más rápido que un vector de punteros.

M10 · Dispersión e Inarmonicidad
Filtros Allpass para Dispersión y Banded Waveguides
Stiffness de piano, f_n = n·f₀·√(1+B·n²), banded DWG para idiófonos

Dispersión y Stiffness

f_n = n·f₀·√(1 + B·n²) donde B ≈ 0.0001–0.01 para piano
H_allpass(z) = (a + z⁻¹)/(1 + a·z⁻¹) a = (1−√B)/(1+√B)
import numpy as np
from scipy.signal import lfilter
from scipy.io import wavfile

def piano_string_ks(freq, duration_s, fs=44100, B=0.001, feedback=0.9998):
    """KS con dispersión de piano (Stiffness B via filtro allpass)."""
    L = int(fs / freq)
    buffer = np.random.randn(L)
    output = np.zeros(int(duration_s * fs))
    ptr = 0
    a_disp = float(np.clip((1 - np.sqrt(B)) / (1 + np.sqrt(B)), -0.999, 0.999))
    allpass_state = 0.0
    for n in range(len(output)):
        current = buffer[ptr]
        next_ptr = (ptr + 1) % L
        avg = feedback * 0.5 * (current + buffer[next_ptr])
        # Filtro allpass para dispersión
        allpass_out = a_disp * avg + allpass_state
        allpass_state = avg - a_disp * allpass_out
        buffer[ptr] = allpass_out
        output[n] = current
        ptr = (ptr + 1) % L
    return output

# C4 con B=0.001 (cuerdas medias piano) vs B=0.01 (cuerdas cortas)
piano_med = piano_string_ks(261.6, 5.0, B=0.001)
piano_hi  = piano_string_ks(261.6, 5.0, B=0.01)
for sig, name in [(piano_med, 'b001'), (piano_hi, 'b01')]:
    s = sig / (np.max(np.abs(sig)) + 1e-8)
    wavfile.write(f'piano_{name}.wav', 44100, (s*32767).astype(np.int16))

Banded Waveguides (para idiófonos con fuerte inarmonicidad): Σ_k [BPF_k * DelayLine_k] donde f_k = frecuencia real del k-ésimo parcial. B crece con diámetro y disminuye con longitud: cuerdas de violín (B≈0.000002) son casi harmónicas; cuerdas de piano corto (B≈0.003) son perceptiblemente inarmónicas.

M11 · Excitación No Lineal
Stick-Slip, Martillo de Piano y Oversampling
Fricción no-lineal, distorsión con oversampling 3×

Stick-Slip y Soft Clipper con Oversampling

Señal → Upsample 3× → Non-Linear → Downsample 3× (con LPF anti-aliasing)
import numpy as np
from scipy.signal import butter, resample_poly

def soft_clipper_cubic(x, threshold=0.5):
    """Soft clipper cúbico: f(x) = x − x³/3 (genera principalmente 3er armónico)."""
    x_norm = x / threshold
    clipped = np.where(
        np.abs(x_norm) <= 1,
        x_norm - x_norm**3 / 3,
        np.sign(x_norm) * (2/3)
    )
    return clipped * threshold * 1.5

def waveguide_with_nl_distortion(freq, duration_s, fs=44100,
                                  distortion_gain=5.0, oversampling=3):
    """KS con soft-clipper en el loop. Oversampling para evitar aliasing."""
    fs_os = fs * oversampling
    L_os = int(fs_os / freq)
    buffer = np.random.randn(L_os)
    output_os = np.zeros(int(duration_s * fs_os))
    ptr = 0
    for n in range(len(output_os)):
        current = buffer[ptr]
        next_ptr = (ptr + 1) % L_os
        avg = 0.998 * 0.5 * (current + buffer[next_ptr])
        driven = avg * distortion_gain
        buffer[ptr] = soft_clipper_cubic(driven) / distortion_gain
        output_os[n] = current
        ptr = (ptr + 1) % L_os
    output = resample_poly(output_os, 1, oversampling)
    return output / (np.max(np.abs(output)) + 1e-8)

clean     = waveguide_with_nl_distortion(110, 2.0, distortion_gain=1.0, oversampling=1)
distorted = waveguide_with_nl_distortion(110, 2.0, distortion_gain=8.0, oversampling=3)

La no-linealidad dentro del loop interactúa con las frecuencias resonantes: genera armónicos que también circulan y generan sus propios armónicos (intermodulación recursiva) → sonido denso y sustain. Sin oversampling 3×, los armónicos generados por el clipper cúbico superan Nyquist → aliasing.

Síntesis Modal

Salida = Σ_k H_k(z) · Excitación (suma de resonadores 2do orden)
H_k(z) = A_k / (1 − 2·r_k·cos(ω_k)·z⁻¹ + r_k²·z⁻²)
Vocal /a/: F1≈800Hz, F2≈1200Hz, F3≈2500Hz
import numpy as np
from scipy.signal import lfilter
from scipy.io import wavfile

def modal_resonator(freq, decay_time, amplitude, fs=44100, duration=2.0):
    n = int(duration * fs)
    omega = 2 * np.pi * freq / fs
    r = 10 ** (-3 / (decay_time * fs))  # Polo del resonador
    b = np.array([amplitude, 0.0])
    a = np.array([1.0, -2*r*np.cos(omega), r**2])
    impulse = np.zeros(n); impulse[0] = 1.0
    return lfilter(b, a, impulse)

# Campana de marimba (C4 = 261.6 Hz) — 4 modos principales
# f₀, ~4f₀, ~9f₀, ~17f₀ (ajustados por tallado)
modes = [(261.6, 3.0, 1.0), (1046.5, 0.8, 0.3),
         (2354.4, 0.3, 0.1), (4447.2, 0.1, 0.05)]
marimba = sum(modal_resonator(*m, fs=44100, duration=3.0) for m in modes)
marimba /= np.max(np.abs(marimba)) + 1e-8
wavfile.write('marimba_modal.wav', 44100, (marimba*32767).astype(np.int16))

# Formant synthesis — vocal /a/ con excitación de glotis
def rosenberg_pulse(T0, fs=44100):
    """Pulso de Rosenberg (modelo de glotis)."""
    n = np.arange(int(T0 * fs))
    t = n / fs / T0
    return np.where(t < 0.6, 0.5*(1 - np.cos(np.pi * t / 0.6)),
                    np.where(t < 1.0, np.cos(np.pi * (t - 0.6) / 0.8), 0.0))

Q1 · D'Alembert y DWG

¿Cuántas operaciones por muestra necesita una DWG de guitarra de 440 Hz a 44.1 kHz, y por qué es tan eficiente comparada con FDTD?

Ver respuesta

Aproximadamente L+4 ≈ 104 operaciones (L=100 samples del buffer + 4 operaciones del loop filter y retraso fraccionario). La DWG reduce el costo de O(Nx × Nt) de FDTD a O(L) porque D'Alembert demuestra que solo se necesita propagar la onda en el punto de pickup, no en todos los puntos del espacio. Con FDTD de 200 puntos espaciales serían ~400 operaciones/sample: la DWG es ~40× más eficiente.

Q2 · Euler Simpléctico

¿Qué pasaría si implementaras el martillo de piano con Euler Explícito en lugar de Simpléctico?

Ver respuesta

El sistema MSD del martillo inyectaría energía artificialmente en cada step: la amplitud de la posición del martillo crecería exponencialmente en lugar de decaer, produciendo un golpe que "explota" en pocos milisegundos. El Euler Explícito expande sistemáticamente la elipse en el espacio de fase (x,v). Para el martillo de piano con Kh=10⁶ N/m y m=0.01 kg: ω₀=10⁴ rad/s. Con fs=44.100 Hz, ω₀·h≈0.227 < 2: solo el Simpléctico es estable.

Q3 · Inarmonicidad del Piano

¿Por qué el piano se afina con "stretch tuning" y cómo se relaciona con el modelado físico?

Ver respuesta

La inarmonicidad (B > 0) hace que los armónicos reales de las cuerdas graves no coincidan con los fundamentales ideales de las cuerdas agudas. El stretch tuning ajusta las frecuencias fundamentales para que las octavas suenen consonantes cuando los armónicos REALES de las cuerdas graves coinciden con los fundamentales REALES de las octavas superiores. Un modelo PM correcto que incluye inarmonicidad debe también aplicar stretch tuning: si C2 tiene B=0.003, su 8vo armónico es ~6 Hz más alto que C5 ideal → C5 sube ~6 Hz en el tuning estirado.

Q4 · Síntesis Conmutada

Si el cuerpo del instrumento tiene una resonancia no-lineal (varía con la amplitud), ¿sigue siendo válida la síntesis conmutada?

Ver respuesta

No: la síntesis conmutada requiere que todos los componentes conmutados sean LTI (lineales e invariantes en el tiempo). Un cuerpo con resonancia no-lineal viola la linealidad: f(A + B) ≠ f(A) + f(B). No se puede pre-convolucionar con una IR única porque la respuesta depende de la amplitud de entrada. El piano tampoco puede usar síntesis conmutada pura: el martillo interacciona de forma no-lineal (múltiples rebotes, rigidez variable), violando la condición LTI.

Q5 · Límite Fundamental del PM

¿Cuál es el límite fundamental del modelado físico que impide reproducir exactamente el sonido de un Stradivarius con un modelo matemático?

Ver respuesta

El problema inverso: para reproducir exactamente un Stradivarius necesitarías conocer todos sus parámetros físicos (forma de cada fibra de madera, distribución de barniz, historia de tensión, temperatura actual) con precisión imposible de medir no-destructivamente. El modelo es tan bueno como la medición de sus parámetros, y medir todos ellos destruiría el instrumento. El mejor PM puede aproximarse a un "violín excelente genérico", no al instrumento específico. Esta es la frontera honesta del modelado físico.
