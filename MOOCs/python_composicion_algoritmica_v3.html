M1 · Audio Basics
Python Audio Basics
Fundamentos de audio digital con Python: samples, FFT, lectura/escritura de audio

Representacion Digital del Audio

  • Sample rate: numero de muestras por segundo. CD = 44 100 Hz, estudio = 48 000/96 000 Hz
  • Bit depth: resolucion de amplitud. 16-bit = 65 536 niveles, 24-bit = 16 M niveles
  • Frame: bloque de samples procesado de golpe (tipicamente 512, 1024 o 2048 samples)
  • Nyquist: la frecuencia maxima representable = sample_rate / 2

Lectura, Escritura y Sintesis

import soundfile as sf
import numpy as np

# Leer archivo WAV
audio, sr = sf.read("sample.wav")
print(f"Samples: {len(audio)}, SR: {sr}, Dur: {len(audio)/sr:.2f}s")

# Generar ondas basicas
SR = 44100

def sine(freq, dur, amp=0.5):
    t = np.linspace(0, dur, int(SR * dur), endpoint=False)
    return amp * np.sin(2 * np.pi * freq * t)

def apply_adsr(signal, sr, a=0.01, d=0.1, s=0.7, r=0.2):
    n = len(signal); a_n = int(a*sr); d_n = int(d*sr); r_n = int(r*sr)
    env = np.ones(n)
    env[:a_n] = np.linspace(0, 1, a_n)
    env[a_n:a_n+d_n] = np.linspace(1, s, d_n)
    env[a_n+d_n:n-r_n] = s
    env[n-r_n:] = np.linspace(s, 0, r_n)
    return signal * env

# Escala Do Mayor
notes = [261.63, 293.66, 329.63, 349.23, 392.0, 440.0, 493.88, 523.25]
scale = np.concatenate([apply_adsr(sine(f, 0.4), SR) for f in notes])
sf.write("c_major_scale.wav", scale, SR)

# FFT y deteccion de pitch
N = 4096; audio_mono = audio if audio.ndim == 1 else audio.mean(axis=1)
fft = np.fft.rfft(audio_mono[:N], n=N)
freqs = np.fft.rfftfreq(N, 1/sr)
dominant_freq = freqs[np.argmax(np.abs(fft))]
print(f"Frecuencia dominante: {dominant_freq:.1f} Hz")

Q1 · Feynman Quiz — FFT y Pitch

¿Por que se aplica una ventana (Hann, Blackman) antes de computar la FFT de una señal de audio? ¿Que artefacto evita?

Ver respuesta

La FFT asume que la señal es periodica dentro de la ventana de analisis. Si la señal no es un multiplo exacto del periodo de la ventana, existe discontinuidad en los bordes (el ultimo sample no se "conecta" suavemente con el primero). Esta discontinuidad genera el "spectral leakage" o fuga espectral: energia de una frecuencia se dispersa a todas las frecuencias vecinas, levantando el suelo de ruido del espectro y dificultando la deteccion de picos. Una ventana como Hann multiplica la señal por una funcion que vale cero en los extremos, eliminando las discontinuidades y concentrando la energia de cada frecuencia en un pico estrecho y bien definido a costa de un leve ensanchamiento del lobulo principal.

Q2 · Feynman Quiz — MIDI vs OSC

¿Cual es la diferencia fundamental entre MIDI y OSC como protocolos de comunicacion musical? ¿En que escenario usarias cada uno?

Ver respuesta

MIDI es un protocolo serial de bajo nivel con mensajes de tipo fijo (note_on, note_off, CC, etc.) de 1-3 bytes, disenado en 1983 para comunicacion entre dispositivos hardware. Tiene resolucion de 7 bits (0-127) y latencia muy baja (<1ms en hardware). OSC (Open Sound Control) es un protocolo de red basado en UDP que permite enviar mensajes arbitrarios con rutas tipo URL (/synth/freq, /note/play) y argumentos de cualquier tipo (float64, string, blobs). OSC se usa cuando necesitas flexibilidad semantica (mensajes personalizados), mayor precision (float64 para frecuencia exacta en Hz), o comunicacion entre software en red (Python↔SuperCollider, Python↔TouchDesigner). MIDI se usa cuando la compatibilidad con hardware es prioritaria o cuando trabajas con DAWs que lo requieren nativamente.

M3 · Caos
Sistemas Caoticos
Mapas logisticos, atractores de Lorenz y bifurcaciones aplicados a parametros musicales

El Mapa Logistico

xₙ₊₁ = r · xₙ · (1 − xₙ) — donde x ∈ (0,1) y r ∈ (0,4]
  • r < 3.0 → converge a punto fijo
  • r 3.0–3.57 → ciclos periodicos (2, 4, 8...)
  • r > 3.57 → comportamiento caotico
  • r = 4.0 → caos completo, ergodico

Mapa Logistico y Atractor de Lorenz → MIDI

import numpy as np; import mido

def logistic_map(r, x0, n):
    x = x0; values = []
    for _ in range(n):
        x = r * x * (1 - x); values.append(x)
    return np.array(values)

def lorenz(sigma=10, rho=28, beta=8/3, dt=0.01, steps=5000):
    x, y, z = 0.1, 0, 0; traj = np.zeros((steps, 3))
    for i in range(steps):
        dx=sigma*(y-x); dy=x*(rho-z)-y; dz=x*y-beta*z
        x+=dx*dt; y+=dy*dt; z+=dz*dt; traj[i]=[x,y,z]
    return traj

# Mapa logistico → notas pentatonicas
pentatonic = [60,62,64,67,69,72,74,76,79,81]
chaos = logistic_map(r=3.9, x0=0.4, n=64)
notes = [pentatonic[int(v*len(pentatonic))%len(pentatonic)] for v in chaos]
vels  = [int(50 + v*70) for v in logistic_map(r=3.7, x0=0.7, n=64)]

out = mido.MidiFile(ticks_per_beat=480)
track = mido.MidiTrack(); out.tracks.append(track)
track.append(mido.MetaMessage('set_tempo', tempo=500000, time=0))
for note, vel in zip(notes, vels):
    track.append(mido.Message('note_on', note=note, velocity=vel, time=0))
    track.append(mido.Message('note_off', note=note, velocity=0, time=240))
out.save("chaos_melody.mid")
print(f"{len(notes)} notas generadas con mapa logistico r=3.9")

Q3 · Feynman Quiz — Caos Musical

¿Por que el mapa logistico genera caos con r > 3.57? ¿Que propiedad matematica lo hace util como generador musical?

Ver respuesta

El mapa logistico xₙ₊₁ = r·xₙ·(1−xₙ) es un sistema dinamico no-lineal. Para r≤3 converge a un punto fijo. Entre 3 y 3.57 entra en cascada de duplicaciones de periodo (2, 4, 8...). Por encima de 3.57 el sistema se vuelve caotico: sensible a condiciones iniciales (efecto mariposa), aperiodico e impredecible a largo plazo aunque deterministico. Para la composicion musical esto significa: (1) la misma regla genera texturas siempre diferentes dependiendo del valor inicial — ideal para parametrizar dinamica, tempo rubato o densidad sin repeticion exacta; (2) la estructura interna del caos (atractores, cuencas) crea correlaciones a corto plazo que el oido percibe como coherencia, a diferencia del ruido blanco puro; (3) el parametro r actua como control de complejidad continuo desde orden (r<3) hasta caos completo (r=4).

M4 · Markov
Cadenas de Markov
Modelos de Markov de 1er y 2do orden para generacion de melodias y progressiones armonicas

Markov de 1er y 2do Orden

import numpy as np; import mido
from collections import defaultdict

def build_markov_2(corpus):
    """Orden 2: estado = (nota_n-1, nota_n)."""
    transitions = defaultdict(list)
    for i in range(len(corpus) - 2):
        transitions[(corpus[i], corpus[i+1])].append(corpus[i+2])
    probs = {}
    for state, nexts in transitions.items():
        unique, counts = np.unique(nexts, return_counts=True)
        probs[state] = (unique.tolist(), (counts/counts.sum()).tolist())
    return probs

def generate_markov_2(table, start_pair, length, temperatura=1.0):
    seq = list(start_pair)
    for _ in range(length - 2):
        state = (seq[-2], seq[-1])
        if state not in table:
            state = list(table.keys())[np.random.randint(len(table))]
        nexts, probs = table[state]
        probs_t = np.array(probs) ** (1.0/temperatura)
        probs_t /= probs_t.sum()
        seq.append(np.random.choice(nexts, p=probs_t))
    return seq

corpus = [60,62,64,65,64,62,60,62,64,67,65,64,62,64,65,67,69,67,65,64,62,60]
table2 = build_markov_2(corpus)
melody = generate_markov_2(table2, (60, 62), length=48)

out = mido.MidiFile(ticks_per_beat=480)
track = mido.MidiTrack(); out.tracks.append(track)
track.append(mido.MetaMessage('set_tempo', tempo=500000, time=0))
for note in melody:
    track.append(mido.Message('note_on', note=note, velocity=75, time=0))
    track.append(mido.Message('note_off', note=note, velocity=0, time=360))
out.save("markov2_melody.mid")
print(f"Melodia Markov ord-2: {len(melody)} notas")

Q4 · Feynman Quiz — Markov Orden 1 vs 2

¿Por que una cadena de Markov de orden 2 suena mas "musical" que una de orden 1? Explica con terminos de memoria y coherencia intervalica.

Ver respuesta

Una cadena de Markov de orden 1 solo recuerda la nota anterior para decidir la siguiente, produciendo saltos intervalicos aleatorios con poca coherencia frastica. El modelo captura solo la distribucion de probabilidad de notas individuales, no los patrones de movimiento. Una cadena de orden 2 recuerda los 2 ultimos eventos, capturando patrones de movimiento (ascendente/descendente) y tendencias que el oido percibe como intencion melodica. En terminos practicos, el orden 2 preserva micro-frases y gestos caracteristicos del corpus de entrenamiento: si el corpus tiende a subir de Do a Re y luego a Mi, el modelo de orden 2 aprendera ese patron de dos pasos. El orden 1 solo sabra que despues de Do puede venir Re con cierta probabilidad, sin saber de donde venia antes de Do.

M5 · L-Systems
L-Systems
Gramaticas de Lindenmayer para estructuras auto-similares, ritmos fractales y melodias recursivas

Gramaticas de Lindenmayer

Un L-System es una tupla G = (V, ω, P) donde V es el alfabeto, ω el axioma inicial y P las reglas de produccion. Cada iteracion reemplaza simultaneamente todos los simbolos aplicables.

L-System Musical

import mido

def lsystem(axiom, rules, iterations):
    result = axiom
    for _ in range(iterations):
        result = ''.join(rules.get(c, c) for c in result)
    return result

def musical_turtle(lstr, start_note=60, interval=2, beat_ticks=240):
    """Interpreta cadena L-System como melodia con tortuga musical."""
    out = mido.MidiFile(ticks_per_beat=480)
    track = mido.MidiTrack(); out.tracks.append(track)
    track.append(mido.MetaMessage('set_tempo', tempo=480000, time=0))
    note = start_note; stack = []
    for sym in lstr[:256]:
        if sym == 'F':
            n = max(36, min(84, note))
            track.append(mido.Message('note_on', note=n, velocity=75, time=0))
            track.append(mido.Message('note_off', note=n, velocity=0, time=beat_ticks))
        elif sym == '+': note = min(note + interval, 84)
        elif sym == '-': note = max(note - interval, 36)
        elif sym == '[': stack.append(note)
        elif sym == ']': note = stack.pop() if stack else note
    out.save("lsystem_melody.mid")

# L-System para ritmos: F=nota, f=silencio
rhythm_rules = {'F': 'F-f-F', 'f': 'ff-F', '-': '-'}
rhythm_str = lsystem('F', rhythm_rules, 4)

# L-System melodico
melodic_rules = {'F': 'FF+F-F', '+': '+', '-': '-', '[': '[', ']': ']'}
melodic_str = lsystem('F+F+F', melodic_rules, 3)
musical_turtle(melodic_str, start_note=60, interval=2)

Q5 · Feynman Quiz — L-System vs Aleatoriedad

¿Que ventaja ofrece un L-System sobre una secuencia ritmica aleatoria para composicion musical? ¿Que caracteristica perceptiva emerge de la auto-similaridad?

Ver respuesta

Los L-Systems generan estructuras auto-similares y jerarquicas mediante reglas de reescritura deterministicas, produciendo patrones ritmicos con coherencia fractal perceptible. Una secuencia aleatoria tiene distribucion uniforme pero carece de estructura interna. El L-System crea relaciones proporcionales entre niveles de escala temporal: el ritmo de la frase refleja el ritmo del motivo (la misma proporcion 2:1 que existe a nivel de corchea/negra puede aparecer a nivel de frase/seccion). El oido percibe esta auto-similaridad como organizacion formal incluso sin haber aprendido explicitamente la regla generadora, porque nuestro sistema auditivo esta entrenado para detectar patrones jerarquicos (silabas, palabras, frases, secciones). La aleatoriedad pura rompe estas expectativas a todos los niveles simultaneamente, resultando en percepcion de "caos sin sentido".

Implementacion con DEAP

import random
from deap import base, creator, tools, algorithms
import numpy as np

# Definir escala objetivo (Do Mayor)
C_MAJOR = [60, 62, 64, 65, 67, 69, 71, 72]

creator.create("FitnessMax", base.Fitness, weights=(1.0,))
creator.create("Individual", list, fitness=creator.FitnessMax)

toolbox = base.Toolbox()
toolbox.register("attr_note", random.choice, C_MAJOR)
toolbox.register("individual", tools.initRepeat, creator.Individual,
                 toolbox.attr_note, n=16)
toolbox.register("population", tools.initRepeat, list, toolbox.individual)

def fitness_melody(individual):
    score = 0.0
    # 1. Notas en la escala
    score += sum(1 for n in individual if n in C_MAJOR) / len(individual) * 40
    # 2. Movimiento preferentemente por grados conjuntos
    intervals = [abs(individual[i+1] - individual[i]) for i in range(len(individual)-1)]
    score += sum(1 for iv in intervals if iv <= 2) / len(intervals) * 30
    # 3. Rango vocal razonable (dentro de una octava)
    span = max(individual) - min(individual)
    score += max(0, 30 - span)
    return (score,)

toolbox.register("evaluate", fitness_melody)
toolbox.register("mate", tools.cxOnePoint)
toolbox.register("mutate", tools.mutUniformInt, low=C_MAJOR[0],
                 up=C_MAJOR[-1], indpb=0.1)
toolbox.register("select", tools.selTournament, tournsize=3)

pop = toolbox.population(n=100)
algorithms.eaSimple(pop, toolbox, cxpb=0.7, mutpb=0.2,
                    ngen=50, verbose=False)
best = tools.selBest(pop, k=1)[0]
print(f"Mejor melodia: {list(best)}")
print(f"Fitness: {best.fitness.values[0]:.1f}/100")

Q6 · Feynman Quiz — Funcion de Fitness

¿Como funciona la funcion de fitness en un AG musical? ¿Que metricas usarias y como las balanceas para evitar convergencia prematura?

Ver respuesta

La funcion de fitness evalua cuantitativamente cuan "buena" es una solucion musical. Metricas tipicas: (1) consonancia intervalica — penalizar intervalos disonantes en posiciones debiles del compas; (2) diversidad ritmica — recompensar variedad de duraciones; (3) rango vocal — penalizar secuencias fuera del rango idiomatico del instrumento; (4) coherencia tonal — recompensar notas pertenecientes a la escala/modo elegido; (5) movimiento por grados conjuntos — favorece stepwise vs saltos grandes. Para evitar convergencia prematura se usan: ponderacion cuidadosa de cada metrica (ninguna debe dominar absolutamente), mantenimiento de diversidad con nichos o sharing, presion de seleccion moderada (torneo de tamaño 2-3 vs seleccion elitista pura), y mutacion suficiente (5-20%) para explorar el espacio. Un fitness demasiado restrictivo produce melodias tecnicamente correctas pero sin caracter.

Sintesis Aditiva, FM y Filtros

import numpy as np; import soundfile as sf

SR = 44100

# Sintesis aditiva: suma de armonicos (serie 1/n)
def additive_synth(freq, dur, n_harmonics=8, amp=0.5):
    t = np.linspace(0, dur, int(SR*dur), endpoint=False)
    signal = np.zeros_like(t)
    for n in range(1, n_harmonics + 1):
        signal += (1.0/n) * np.sin(2*np.pi*freq*n*t)
    return amp * signal / np.max(np.abs(signal))

# Sintesis FM
def fm_synth(carrier_freq, mod_ratio, mod_index, dur, amp=0.5):
    t = np.linspace(0, dur, int(SR*dur), endpoint=False)
    mod_freq = carrier_freq * mod_ratio
    modulator = mod_index * np.sin(2*np.pi*mod_freq*t)
    carrier   = amp * np.sin(2*np.pi*carrier_freq*t + modulator)
    return carrier

# Filtro paso bajo con scipy
from scipy.signal import butter, filtfilt

def lowpass_filter(signal, cutoff_hz, sr=44100, order=4):
    nyq = sr / 2
    Wn = cutoff_hz / nyq
    b, a = butter(order, Wn, btype='low')
    return filtfilt(b, a, signal)

# Generar y exportar
note_additive = additive_synth(261.63, 2.0)  # Do4, 2 segundos
note_fm = fm_synth(440, mod_ratio=2.0, mod_index=3.0, dur=2.0)  # La4 FM
note_filtered = lowpass_filter(note_fm, cutoff_hz=2000)

import numpy as np
scale_audio = np.concatenate([note_additive, note_fm, note_filtered])
sf.write("synthesis_demo.wav", scale_audio, SR)
print("Demos de sintesis guardados: synthesis_demo.wav")

Q8 · Feynman Quiz — Sintesis FM

¿Como funciona la sintesis FM y que parametros determinan el timbre resultante? ¿Por que un mod_index alto genera timbre mas brillante?

Ver respuesta

La sintesis FM (Frecuency Modulation) usa un oscilador modulador para variar la frecuencia instantanea del oscilador portador: output = sin(2π·f_c·t + I·sin(2π·f_m·t)). Los parametros clave son: carrier_freq (f_c) determina la frecuencia percibida; mod_ratio (f_m/f_c) determina la relacion armonica de los parciales generados (ratios enteros = timbre armonico, ratios irracionales = timbre inarmonico/metalico); mod_index (I) determina la amplitud de la modulacion y directamente el numero y amplitud de los parciales laterales (sidebands) generados. Un mod_index alto genera mas sidebands con mayor amplitud segun los coeficientes de Bessel Jₙ(I): cuanto mayor es I, mayor es n para el que Jₙ(I) es significativo, resultando en un espectro mas rico y brillante con energia distribuida a frecuencias altas. Con I=0 se obtiene un seno puro; con I=3 ya hay sidebands hasta el 6° orden con amplitud audible.

Q10 · Feynman Quiz — FoxDot vs TidalCycles

¿Cual es la diferencia principal entre FoxDot y TidalCycles para live coding musical? ¿Que paradigma de tiempo usa cada uno?

Ver respuesta

FoxDot es Python puro con backend SuperCollider, usando un modelo de tiempo basado en el Clock de Python: cada Player (d1, s1, b1) es un objeto que recibe patrones como listas o Patterns de Python y genera eventos en el servidor SuperCollider en momentos futuros calculados por el scheduler. La diferencia conceptual clave: FoxDot piensa en "esta nota en este momento futuro", TidalCycles piensa en "esta funcion del tiempo para siempre". TidalCycles facilita composicion polirritmica compleja con mini-notation concisa; FoxDot es mas natural para programadores Python que quieren usar su lenguaje habitual.

LSTM para Generacion de Melodias

import numpy as np

# Arquitectura LSTM basica para prediccion de notas
# (usando numpy para mostrar el concepto sin TF/PyTorch)

class SimpleLSTM:
    """LSTM simplificado para demostracion conceptual."""
    def __init__(self, vocab_size, hidden_size):
        self.vocab_size = vocab_size
        self.hidden_size = hidden_size
        # En implementacion real: pesos entrenados con Adam, backprop
        self.W_ih = np.random.randn(4*hidden_size, vocab_size) * 0.01
        self.W_hh = np.random.randn(4*hidden_size, hidden_size) * 0.01
        self.b = np.zeros(4*hidden_size)

    def forward(self, x_onehot, h, c):
        # Puertas LSTM: forget, input, gate, output
        gates = self.W_ih @ x_onehot + self.W_hh @ h + self.b
        f, i, g, o = np.split(gates, 4)
        f = 1/(1+np.exp(-f)); i = 1/(1+np.exp(-i))
        g = np.tanh(g);       o = 1/(1+np.exp(-o))
        c_new = f * c + i * g
        h_new = o * np.tanh(c_new)
        return h_new, c_new

def sample_with_temperature(logits, temperature=1.0):
    """Temperatura: <1 = mas conservador, >1 = mas aleatorio"""
    logits = logits / temperature
    probs = np.exp(logits - np.max(logits))
    probs /= probs.sum()
    return np.random.choice(len(probs), p=probs)

Q11 · Feynman Quiz — LSTM y Temperatura

¿Como controla la temperatura de muestreo la creatividad vs coherencia de una LSTM musical? ¿Que sucede matematicamente con los logits?

Ver respuesta

La temperatura divide los logits (scores pre-softmax) de la LSTM antes de aplicar softmax para obtener probabilidades. Con T<1 (ej. T=0.5): los logits se dividen por 0.5, amplificando las diferencias entre probabilidades → la distribucion se vuelve mas "peaky" y el modelo elige casi siempre la nota mas probable → generacion conservadora, predecible, tonalmente segura pero repetitiva. Con T>1 (ej. T=1.5): los logits se dividen por 1.5, aplastando las diferencias entre probabilidades → la distribucion se aplana → mayor probabilidad de elegir notas sorprendentes o inesperadas → generacion mas creativa pero potencialmente incoherente o fuera de estilo. Con T=1: comportamiento por defecto del modelo. En composicion musical, T=0.8-1.2 suele dar el mejor equilibrio entre coherencia estilistica y variedad sorprendente.

Q12 · Feynman Quiz — MusicVAE vs MelodyRNN

¿Cual es la diferencia principal entre MusicVAE y MelodyRNN de Magenta? ¿Que ventaja ofrece la interpolacion latente de MusicVAE?

Ver respuesta

MelodyRNN es un modelo autoregresivo (LSTM) entrenado para predecir la siguiente nota dado el contexto previo; genera secuencias de izquierda a derecha y es bueno para continuacion y completado. El modelo solo puede generar "hacia adelante" a partir de un inicio. MusicVAE es un Variational Autoencoder que aprende un espacio latente continuo de melodias completas: el encoder mapea una melodia a un punto en el espacio latente Z (vector de ~256 dimensiones), y el decoder mapea cualquier punto Z a una melodia. La ventaja de la interpolacion latente es que permite navegar suavemente entre dos melodias: tomando dos puntos Z1 y Z2 en el espacio latente y generando puntos intermedios Z1 + t*(Z2-Z1) para t∈[0,1], obtenemos melodias que combinan caracteristicas de ambas de forma gradual y musicalmente coherente. Esto no es posible con MelodyRNN, que no tiene nocion de "melodia global" sino de "siguiente nota probable".

Q13 · Feynman Quiz — Latencia MIDI Virtual

¿Por que la latencia de un puerto MIDI virtual en Python puede ser mayor que la de hardware MIDI DIN-5? ¿Como la minimizas para produccion profesional?

Ver respuesta

Un puerto MIDI virtual pasa por varias capas de software: (1) la libreria Python (rtmidi/mido) que llama al driver del SO; (2) el driver del SO (IAC en macOS, ALSA/JACK en Linux) que enruta el mensaje; (3) el buffer del DAW receptor que introduce su propio buffer de latencia. El hardware MIDI DIN-5 tiene latencia determinista de ~1ms (31.25kbps, 1 byte = 320µs). El software introduce jitter variable (1-20ms tipico). Para minimizarla: (1) usar rtmidi directamente (evitar capas intermedias como python-osc); (2) medir y compensar el offset con timestamps de perf_counter(); (3) usar JACK en Linux con prioridad RT para el proceso Python; (4) evitar operaciones bloqueantes (I/O, sleep) en el thread MIDI; (5) en macOS, habilitar el IAC Driver en modo "low latency" en Audio MIDI Setup; (6) enviar mensajes con anticipacion calculada para que lleguen exactamente en el beat correcto.

Q1 · Markov Orden 1 vs 2

¿Por que una cadena de Markov de orden 2 suena mas "musical" que una de orden 1? Explica con el concepto de memoria y coherencia frastica.

Ver respuesta

Una cadena de Markov de orden 1 solo recuerda la nota anterior para decidir la siguiente, produciendo saltos intervalicos aleatorios con poca coherencia frastica. El modelo captura solo la distribucion de probabilidad de notas individuales. Una cadena de orden 2 recuerda los 2 ultimos eventos, capturando patrones de movimiento (ascendente/descendente) y tendencias que el oido percibe como intencion melodica. Preserva micro-frases y gestos caracteristicos del corpus de entrenamiento. En terminos de informacion: el orden 2 tiene memoria condicional P(nota_n | nota_{n-1}, nota_{n-2}), capturando el contexto de los dos ultimos pasos, lo que permite modelar tensiones de leading-tone, movimientos por grado conjunto y patrones de arco melodico que el oido asocia con musica tonal.

Q2 · Fitness en Algoritmos Geneticos

¿Como funciona la funcion de fitness en un AG musical y que metricas usarias para evaluar calidad melodica? ¿Que es convergencia prematura?

Ver respuesta

La funcion de fitness evalua cuantitativamente cuan "buena" es una melodia. Metricas tipicas: (1) consonancia intervalica — penalizar intervalos disonantes en posiciones debiles; (2) coherencia tonal — recompensar notas pertenecientes a la escala elegida; (3) movimiento por grados conjuntos — favorece stepwise vs saltos grandes; (4) rango vocal razonable — penalizar secuencias fuera del rango idiomatico; (5) diversidad ritmica — recompensar variedad de duraciones. La convergencia prematura ocurre cuando la poblacion pierde diversidad genetica rapidamente porque unos pocos individuos muy aptos dominan la seleccion en pocas generaciones, resultando en que el AG queda atrapado en un optimo local. Se evita con: presion de seleccion moderada (torneo de tamaño 2-3), mutacion suficiente (5-20%), sharing/niching para mantener diversidad, y fitness multi-objetivo que penalice la similitud entre individuos de la misma poblacion.

Q3 · L-System vs Aleatoriedad

¿Que ventaja ofrecen los L-Systems sobre secuencias aleatorias para generacion ritmica? ¿Que es la auto-similaridad fractal en el contexto musical?

Ver respuesta

Los L-Systems generan estructuras auto-similares y jerarquicas mediante reglas de reescritura deterministicas, produciendo patrones ritmicos con coherencia fractal perceptible. Una secuencia aleatoria tiene distribucion uniforme pero carece de estructura interna. El L-System crea relaciones proporcionales entre niveles de escala temporal: el ritmo de la frase refleja el ritmo del motivo. La auto-similaridad fractal en musica significa que la misma proporcion ritmica que aparece a nivel de corchea/negra puede reaparecer a nivel de frase/seccion. El oido percibe esta organizacion como coherencia formal incluso sin conocer la regla, porque el sistema auditivo humano esta entrenado para detectar jerarquias temporales (silabas, palabras, frases). La aleatoriedad rompe estas expectativas a todos los niveles simultaneamente.

Q4 · Temperatura de Muestreo en LSTM

¿Como controla la temperatura de muestreo la creatividad vs coherencia de una LSTM musical? Explica matematicamente que le sucede a los logits con T=0.5 y T=1.5.

Ver respuesta

La temperatura divide los logits (scores pre-softmax) antes de aplicar softmax. Con T=0.5: logits/0.5 amplifica las diferencias → distribucion mas "peaky" → el modelo elige casi siempre la nota mas probable → generacion conservadora, predecible, tonalmente segura pero repetitiva. Con T=1.5: logits/1.5 aplasta las diferencias → distribucion mas plana → mayor probabilidad de elegir notas sorprendentes → generacion mas creativa pero potencialmente incoherente. Matematicamente: softmax(logits/T) concentra o aplana la distribucion de probabilidad. Para T→0 se obtiene seleccion greedy (siempre la nota maxima). Para T→∞ se obtiene distribucion uniforme (completamente aleatoria). En composicion musical, T=0.8-1.2 suele dar el mejor equilibrio entre coherencia estilistica y variedad sorprendente.

Q5 · MusicVAE vs MelodyRNN

¿Cual es la diferencia principal entre MusicVAE y MelodyRNN de Magenta? ¿Que ventaja creativa especifica ofrece la interpolacion en el espacio latente?

Ver respuesta

MelodyRNN es un modelo autoregresivo (LSTM) que genera notas de izquierda a derecha: P(nota_n | nota_{n-1}, nota_{n-2}, ...). Es bueno para continuacion pero no puede "ver" la forma global. MusicVAE es un Variational Autoencoder que aprende un espacio latente continuo de melodias completas: el encoder mapea una melodia a un punto en Z (vector de ~256 dimensiones), el decoder mapea cualquier Z a una melodia. La ventaja de interpolacion latente: tomando Z1 (melodia A) y Z2 (melodia B) y generando puntos intermedios Z1 + t*(Z2-Z1) para t∈[0,1], se obtienen melodias que combinan caracteristicas de ambas de forma gradual. Esto permite crear puentes musicales entre temas, morphing timerico, o explorar sistematicamente el espacio de melodias posibles. No es posible con MelodyRNN que no tiene concepto de "melodia global" sino solo de "siguiente nota probable".
