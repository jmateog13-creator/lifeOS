class AmazighRoot:
    def __init__(self, consonants: list):
        self.C = consonants  # e.g., ['k', 'r', 'z']

    def apply_template(self, template: str) -> str:
        result = []; c_idx = 0
        for char in template:
            if char == '_':
                result.append(self.C[c_idx]); c_idx += 1
            else:
                result.append(char)
        return ''.join(result)

M5 · Morfosintaxis
Optimality Theory
Constraints, tableaux, ranking y predicción de outputs sin derivaciones intermedias

Arquitectura OT

OT elimina reglas ordenadas secuencialmente. El generador GEN produce todos los candidatos posibles; EVAL los filtra en una pasada usando el ranking jerárquico de constraints. El output es el candidato que viola menos constraints altamente rankeados.

MóduloFunciónAnalogía computacional
GENProduce todos los candidatos posibles desde el inputGenerador exhaustivo
CONConjunto de constraints universalesFunción de coste multidimensional
EVALFiltra por ranking jerárquicoBúsqueda del óptimo por ranking

Constraints clave para Tamazight: MAX-IO (no elisión), DEP-IO (no epéntesis), IDENT-IO[place] (preservar lugar de articulación).

M8 · Fonética Computacional
Escala Bark y Psicoacústica
Conversión Hz→Bark, corrección Traunmüller, HDI+ROPE para significancia
Bark (Traunmüller): z = 26.81·f/(1960+f) − 0.53
z' = z + 0.15·(2−z) si z < 2.0
z' = z + 0.22·(z−20.1) si z > 20.1

Implementación de Conversión Bark

def hz_to_bark(f: float) -> float:
    if f <= 0: return 0.0
    z = 26.81 * f / (1960 + f) - 0.53
    if z < 2.0:    z += 0.15 * (2.0 - z)
    elif z > 20.1: z += 0.22 * (z - 20.1)
    return z

# ROPE para significancia clínica: ±0.5 Bark
# Si ΔBark > 0.5 → efecto lingüísticamente significativo
delta_bark = hz_to_bark(1350) - hz_to_bark(820)
print(f"ΔBark = {delta_bark:.3f}")  # ~1.85 → SIGNIFICATIVO

La decisión HDI+ROPE compara el intervalo de alta densidad del posterior bayesiano con la región de equivalencia práctica (±0.5 Bark). Si el HDI cae fuera del ROPE: efecto real.

from scipy.signal import butter, filtfilt
import numpy as np

def filter_ema_signal(data: np.ndarray, fs=1250, cutoff=20) -> np.ndarray:
    """Butterworth LP 20Hz para señales EMA (elimina ruido de alta frecuencia)."""
    b, a = butter(4, cutoff / (fs / 2), btype='low')
    return filtfilt(b, a, data, axis=0)

from skimage.morphology import thin
from skimage.feature import corner_harris, corner_peaks
from sklearn.svm import SVC
from sklearn.model_selection import cross_val_score
import numpy as np

def extract_tifinagh_features(glyph_binary: np.ndarray) -> np.ndarray:
    """
    Pipeline Zhang-Wang + keypoints para features de OCR Tifinagh.
    1. Skeletonización: reduce glifo a esqueleto de 1 píxel
    2. Branch points: vecinos >= 3 en el esqueleto
    3. End points: vecinos == 1 en el esqueleto
    """
    skeleton = thin(glyph_binary)
    peaks = corner_peaks(corner_harris(skeleton.astype(float)), min_distance=3)
    n_branches = 0; n_ends = 0
    for (r, c) in peaks:
        neighborhood = skeleton[max(0,r-1):r+2, max(0,c-1):c+2].sum() - 1
        if neighborhood >= 3: n_branches += 1
        elif neighborhood == 1: n_ends += 1
    return np.array([n_branches, n_ends, skeleton.sum(),
                     glyph_binary.mean(), peaks.shape[0]])

HSGP — Hilbert Space GP para Trayectorias de Formantes

El GP exacto requiere invertir K de N×N. Para EMA a 1250Hz durante 5min, N=375000 → imposible. El HSGP proyecta el GP sobre las primeras m eigenfunciones del Laplaciano. Con m=8, captura >99% de la varianza de trayectorias articulatorias suaves.

import pymc as pm
import numpy as np

def build_hsgp_spread_model(times, f2_obs, m=8):
    with pm.Model() as model:
        # Hiperparámetros del GP
        ls = pm.HalfNormal("ls", sigma=0.05)      # longitud de escala (s)
        eta = pm.HalfNormal("eta", sigma=200)     # amplitud (Hz)
        sigma = pm.HalfNormal("sigma", sigma=30)  # ruido de medida

        # HSGP con m bases en el intervalo [0, max(times)]
        L = max(times) * 1.2
        cov = eta**2 * pm.gp.cov.ExpQuad(1, ls=ls)
        gp = pm.gp.HSGP(m=[m], L=[L], cov_func=cov)
        f = gp.prior("f", X=times[:, None])

        # Likelihood
        pm.Normal("obs", mu=f, sigma=sigma, observed=f2_obs)
    return model

import librosa, numpy as np

def analyze_amazigh_music(wav_path: str):
    y, sr = librosa.load(wav_path, sr=22050)

    # 1. Espectrograma del Bendir
    D = librosa.amplitude_to_db(np.abs(librosa.stft(y)), ref=np.max)

    # 2. Análisis de IOI (Inter-Onset Intervals) para hemiola
    onset_frames = librosa.onset.onset_detect(y=y, sr=sr)
    onset_times = librosa.frames_to_time(onset_frames, sr=sr)
    ioi = np.diff(onset_times)

    # 3. Pitch microtonal con YIN (resolución ~1 cent)
    f0, voiced_flag, _ = librosa.pyin(y, fmin=60, fmax=800)
    f0_voiced = f0[voiced_flag]

    return {"ioi_mean_ms": np.mean(ioi)*1000,
            "f0_median_hz": np.nanmedian(f0_voiced),
            "n_onsets": len(onset_times)}

# Hemiola: relación 3:2 entre niveles métricos
# IOI típico Ahwash 120-150 BPM → 400-500ms/beat

La hemiola amazigh crea un polirritmo 3:2 entre el Tbel (tambor) y el coro vocal. El análisis de IOI detecta esta relación matemáticamente: si IOI_A / IOI_B ≈ 1.5, existe hemiola.

Robustez del Ahwash como Sistema de Transmisión Cultural

El Ahwash implementa tres mecanismos de robustez simultáneos:

  • Redundancia masiva: 150 nodos transmiten el mismo contenido simultáneamente.
  • Corrección de errores distribuida: Si un participante comete un error, los otros 149 lo corrigen por presión auditiva y social.
  • Memoria procedural motora: El ritmo del palmar y la melodía del coro se almacenan en memoria motora — extraordinariamente persistente (como andar en bicicleta).

Resultado: un protocolo de transmisión cultural que ha sobrevivido 4000 años sin escritura.

import numpy as np

def kuramoto_order_parameter(phases: np.ndarray) -> float:
    """Calcula R (parámetro de orden de Kuramoto) dado array de fases."""
    return abs(np.mean(np.exp(1j * phases)))

# Simulación: 150 participantes con frecuencias ligeramente distintas
N = 150; K = 3.0; dt = 0.01; T = 30
omega = np.random.normal(2*np.pi*2.0, 0.1, N)  # ~2 Hz ± variación
theta = np.random.uniform(0, 2*np.pi, N)
for _ in range(int(T/dt)):
    coupling = (K/N) * np.sum(np.sin(theta[:,None] - theta[None,:]), axis=1)
    theta += (omega + coupling) * dt
print(f"R final = {kuramoto_order_parameter(theta):.3f}")

Examen Feynman — Explica sin notas

Q1 · Morfología No-Concatenativa

¿Por qué la morfología no-concatenativa del Tamazight es un reto computacional único para el NLP y qué tipo de formalismo matemático requiere?

Ver respuesta

Los stemmers concatenativos (Snowball, Porter) asumen que los morfemas son cadenas lineales. En Tamazight, las vocales no se concatenan — se proyectan simultáneamente sobre el esqueleto consonántico. La raíz K-R-Z ocupa nodos C del esqueleto; el patrón {a_a} ocupa nodos V. No hay orden temporal de concatenación: ambos se superponen sobre la misma estructura. Computacionalmente se requieren gramáticas bidimensionales o de árboles adjuntos (TAG), no FSTs lineales. Por eso cualquier NLP estándar aplicado al Tamazight produce outputs sin sentido.

Q2 · F₂ como Formante Diagnóstico

¿Por qué el F₂ es el formante diagnóstico de la faringalización y no el F₁ o el F₃?

Ver respuesta

F₁ es controlado por la apertura mandibular — la faringalización no mueve la mandíbula, así que F₁ cambia poco. F₃ está controlado por el cuerpo lingual y labialización — la señal es débil. F₂ es directamente sensible a la posición anteroposterior del dorso lingual: un TBO retraído aumenta el volumen de la cavidad posterior y disminuye F₂ de forma dramática (−400 a −600 Hz para /tˤ/). Además, F₂ está en un rango (700-2500 Hz) donde el sistema auditivo tiene alta resolución, haciendo el contraste perceptivamente saliente.

Q3 · Optimality Theory sin Reglas Ordenadas

¿Por qué la Optimality Theory predice outputs sin reglas ordenadas secuencialmente? ¿Qué ventaja computacional ofrece?

Ver respuesta

Los modelos SPE aplican reglas secuencialmente: el orden importa y produce paradojas de bleeding/feeding. OT elimina este problema: GEN produce todos los candidatos simultáneamente, y EVAL los filtra en una única pasada usando el ranking jerárquico. El output es el candidato que gana el torneo de restricciones. La ventaja computacional: el problema se reduce a buscar el óptimo en un conjunto de candidatos mediante un algoritmo de ranking — sin el colapso de parsing de gramáticas ordenadas.

Q4 · HSGP vs GP Exacto

¿Por qué el HSGP es preferible a un GP exacto para modelar trayectorias EMA en estudios de faringalización?

Ver respuesta

Un GP exacto requiere invertir la matriz K de N×N. Para señal EMA a 1250 Hz durante 5 min, N=375000 → invertir requiere O(N³) = ~5×10¹⁶ flops, imposible en cualquier hardware disponible. El HSGP proyecta el GP sobre las primeras m eigenfunciones del Laplaciano (m=8). Solo necesita m×N = 3 millones de operaciones — factible en segundos. Para trayectorias articulatorias suaves (sin discontinuidades), m=8 captura >99% de la varianza: el trade-off es óptimo por diseño.

Q5 · Ahwash como Sistema sin Escritura

¿Por qué el Ahwash de 150 participantes es un sistema robusto de preservación cultural sin escritura? Nombra los tres mecanismos de robustez.

Ver respuesta

El Ahwash implementa: (1) Redundancia masiva: 150 nodos transmiten simultáneamente — para que se pierda información, deben fallar todos a la vez. (2) Corrección de errores distribuida en tiempo real: un error rítmico o léxico es corregido instantáneamente por presión auditiva y social de los 149 restantes — es un ECC distribuido. (3) Embodiment en memoria motora: el ritmo y melodía se almacenan en memoria procedural (no declarativa) — extremadamente persistente, como montar en bicicleta. Los tres mecanismos combinados explican 4000 años de persistencia sin escritura.
