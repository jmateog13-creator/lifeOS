M1 · Óptica
Óptica de Proyección y Cálculo de Lúmenes
Throw Ratio, distancia, footlamberts, ganancia de pantalla
Throw Ratio: R = D / W (adimensional)
Lúmenes = (foot-lamberts × Área_ft²) / Ganancia
1 foot-lambert = 3.426 nits = 10.764 Lux
Estándar cine digital: 16 ftL | Teatro: 4-8 ftL | Exterior: 50+ ftL

Calculador de Proyección

def calculate_projection(throw_ratio: float, image_width_m: float,
                          desired_ftl: float = 16, screen_gain: float = 1.0) -> dict:
    """
    throw_ratio: R = D/W (ej: 1.5 = 1.5m de distancia por 1m de imagen)
    desired_ftl: foot-lamberts objetivo (16 = cine, 50 = exterior noche)
    screen_gain: blanco mate=1.0, gris=0.8, holográfico=2.5
    """
    distance_m      = throw_ratio * image_width_m
    image_height_m  = image_width_m * 9 / 16           # asumiendo 16:9
    area_m2         = image_width_m * image_height_m
    area_ft2        = area_m2 * 10.764                  # 1m² = 10.764 ft²
    lumens_needed   = (desired_ftl * area_ft2) / screen_gain

    return {
        "distancia_m":      round(distance_m, 2),
        "area_m2":          round(area_m2, 2),
        "lumens_proyector": round(lumens_needed, 0),
        "nits":             round(desired_ftl * 3.426, 1)
    }

# Fachada 8m, throw ratio 1.5, exterior nocturno 50ftL, concreto claro gain=0.8
params = calculate_projection(1.5, 8, 50, 0.8)
# → distancia: 12m, lúmenes: ~38,000 ANSI lumens

M2 · Homografía
Homografía — Geometría Proyectiva 2D
Transformación 3×3, 8 DOF, DLT, RANSAC, cv2.findHomography
H: [x' y' w']ᵀ = H · [x y 1]ᵀ → x_proj = x'/w', y_proj = y'/w'
DLT: cada par de puntos aporta 2 ecuaciones → sistema Ah=0 → SVD
RANSAC: itera subsets mínimos (4 pts), cuenta inliers con umbral

DLT — Direct Linear Transform

import numpy as np

def compute_homography_dlt(src_pts: np.ndarray, dst_pts: np.ndarray) -> np.ndarray:
    """
    DLT mínimo: 4 correspondencias de puntos (N >= 4).
    src_pts, dst_pts: [N, 2] float
    """
    N = len(src_pts)
    A = []
    for (x, y), (xp, yp) in zip(src_pts, dst_pts):
        A.append([-x, -y, -1,  0,  0,  0, x*xp, y*xp, xp])
        A.append([ 0,  0,  0, -x, -y, -1, x*yp, y*yp, yp])
    A = np.array(A)

    _, _, Vt = np.linalg.svd(A)
    H = Vt[-1].reshape(3, 3)
    H /= H[2, 2]
    return H

def apply_homography(H: np.ndarray, pts: np.ndarray) -> np.ndarray:
    pts_h = np.hstack([pts, np.ones((len(pts), 1))])
    pts_t = (H @ pts_h.T).T
    return pts_t[:, :2] / pts_t[:, 2:3]

RANSAC con OpenCV

import cv2
import numpy as np

def robust_homography(src_pts: np.ndarray, dst_pts: np.ndarray,
                       threshold: float = 5.0) -> tuple:
    """
    RANSAC elimina correspondencias outlier (detectadas incorrectamente).
    threshold: distancia máxima en píxeles para ser inlier
    """
    src = src_pts.reshape(-1,1,2).astype(np.float32)
    dst = dst_pts.reshape(-1,1,2).astype(np.float32)

    H, mask = cv2.findHomography(src, dst, cv2.RANSAC, threshold)
    inliers = mask.ravel().astype(bool)
    print(f"Inliers: {inliers.sum()}/{len(src_pts)}")
    return H, inliers

def warp_frame(frame: np.ndarray, H: np.ndarray, output_size: tuple) -> np.ndarray:
    # cv2.warpPerspective usa Intel IPP / AVX-512 internamente
    return cv2.warpPerspective(frame, H, output_size, flags=cv2.INTER_LINEAR)

Superficies Bézier y Warping de Malla

P(u,v) = ΣᵢΣⱼ B(m,i,u) · B(n,j,v) · Pᵢⱼ

Superficie Bézier Bicúbica

import numpy as np
from scipy.special import comb

def bernstein(n: int, k: int, t: float) -> float:
    """Base de Bernstein B(n,k,t)."""
    return comb(n, k, exact=True) * (t**k) * ((1-t)**(n-k))

def bezier_surface(control_pts: np.ndarray, u: float, v: float) -> np.ndarray:
    """
    Evalúa superficie Bézier tensorial en (u,v) ∈ [0,1]².
    control_pts: [m+1, n+1, 2] — grilla de puntos de control 2D
    """
    m, n = control_pts.shape[0]-1, control_pts.shape[1]-1
    result = np.zeros(2)
    for i in range(m+1):
        for j in range(n+1):
            result += bernstein(m,i,u) * bernstein(n,j,v) * control_pts[i,j]
    return result

def generate_bezier_mesh(control_pts: np.ndarray,
                          resolution: int = 20) -> np.ndarray:
    """
    Genera malla de puntos para renderizado.
    Retorna: [resolution, resolution, 2]
    """
    us = np.linspace(0, 1, resolution)
    vs = np.linspace(0, 1, resolution)
    mesh = np.zeros((resolution, resolution, 2))
    for i, u in enumerate(us):
        for j, v in enumerate(vs):
            mesh[i, j] = bezier_surface(control_pts, u, v)
    return mesh

M4 · Blending
Edge Blending — Fusión de Proyectores
Curva de blend no-lineal, corrección gamma, zona de overlap
f(x) = a · (2x)^p para x ∈ [0, 0.5]
x: posición normalizada en la zona de overlap [0=borde, 0.5=centro]
p: exponente de gamma (típico: 1.8–2.5), a: factor de escala
Aplicar ANTES de gamma display para linearidad correcta

Implementación de Edge Blending

import numpy as np

def edge_blend_curve(x: np.ndarray, power: float = 2.0, gain: float = 1.0) -> np.ndarray:
    """
    Curva de edge blending para zona de overlap.
    x: array de valores [0, 1] — 0=borde externo, 1=interior pleno
    power: exponente (más alto = transición más abrupta)

    La curva es: f(x) = gain * (x)^power
    Nota: la fórmula canónica es a*(2x)^p para x ∈ [0,0.5]
    """
    return gain * np.power(np.clip(x, 0, 1), power)

def generate_blend_mask(width: int, overlap_px: int,
                         power: float = 2.0, side: str = 'right') -> np.ndarray:
    """
    Genera máscara de blend para un proyector.
    width: ancho total del proyector en píxeles
    overlap_px: ancho de la zona de overlap
    side: 'left' o 'right' — qué lado se desvanece
    """
    mask = np.ones(width, dtype=np.float32)

    if side == 'right':
        blend_region = np.linspace(1, 0, overlap_px)
        blend_region = edge_blend_curve(blend_region, power)
        mask[-overlap_px:] = blend_region
    else:  # left
        blend_region = np.linspace(0, 1, overlap_px)
        blend_region = edge_blend_curve(blend_region, power)
        mask[:overlap_px] = blend_region

    # Aplicar corrección gamma ANTES de la pantalla
    # (la pantalla aplica gamma display ~ 2.2)
    # Si no se corrige: la zona de blend se ve demasiado oscura
    display_gamma = 2.2
    mask_linear = mask  # la máscara debe ser lineal
    return mask_linear[:, np.newaxis]  # [width, 1] para broadcast

# Ejemplo: dos proyectores de 1920px con 200px de overlap
mask_proj1 = generate_blend_mask(1920, 200, power=2.0, side='right')

Black Level Matching

En la zona de overlap, ambos proyectores proyectan simultáneamente. El negro de un proyector (que no es 0 nits sino ~0.005-0.05 nits) se suma al negro del otro, creando una banda más brillante que el resto. Solución: elevar el black level de los proyectores fuera del overlap para igualarlo.

def black_level_match(overlap_mask: np.ndarray,
                       projector_black_nits: float = 0.02) -> np.ndarray:
    """
    Añade black level en zonas de no-overlap para compensar el
    brillo adicional de la zona de overlap (dos proyectores superpuestos).
    """
    # En overlap: 2 × black_nits → fuera: 1 × black_nits
    # → fuera del overlap necesita +black_nits adicionales
    outside_overlap = 1.0 - overlap_mask
    black_correction = outside_overlap * projector_black_nits
    return black_correction

Análisis FFT para Audio-Reactividad

import numpy as np
from scipy.fft import rfft, rfftfreq

def extract_frequency_bands(audio_chunk: np.ndarray,
                              sample_rate: int = 44100) -> dict:
    """
    Extrae energía por bandas de frecuencia para video mapping reactivo.
    audio_chunk: bloque de audio [N samples] float32 [-1,1]
    """
    N = len(audio_chunk)

    # Ventana Hann para reducir spectral leakage
    window = np.hanning(N)
    X = rfft(audio_chunk * window)
    freqs = rfftfreq(N, 1/sample_rate)

    # Magnitud normalizada
    magnitude = np.abs(X) / N * 2  # 2 por FFT de una cara

    def band_energy(f_low: float, f_high: float) -> float:
        mask = (freqs >= f_low) & (freqs < f_high)
        return float(np.sqrt(np.mean(magnitude[mask]**2)))  # RMS de la banda

    return {
        "sub_bass":   band_energy(20, 80),     # bombos, bajos profundos
        "bass":       band_energy(80, 300),    # línea de bajo
        "low_mid":    band_energy(300, 1000),  # cuerpo de instrumentos
        "mid":        band_energy(1000, 4000), # voz, melodía
        "high":       band_energy(4000, 20000) # brillo, platillos
    }

Proyector-Cámara Loop para Calibración

import cv2
import numpy as np

class ProjectorCalibrator:
    """
    Calibración automática usando structured light:
    1. Proyector emite patrones (checkerboard / Gray code)
    2. Cámara captura los patrones en la superficie
    3. Se establece la correspondencia proyector-pixel → superficie-pixel
    """

    def __init__(self, projector_resolution=(1920, 1080)):
        self.proj_w, self.proj_h = projector_resolution
        self.camera = cv2.VideoCapture(0)

    def project_and_capture_checkerboard(self, pattern_size=(9,6)):
        """
        Proyecta un checkerboard y lo captura con la cámara.
        """
        # Crear checkerboard en espacio del proyector
        sq_size = 80  # píxeles por cuadrado
        cb = np.zeros((self.proj_h, self.proj_w), dtype=np.uint8)
        for row in range(pattern_size[1]):
            for col in range(pattern_size[0]):
                if (row + col) % 2 == 0:
                    y1, y2 = row*sq_size, (row+1)*sq_size
                    x1, x2 = col*sq_size, (col+1)*sq_size
                    cb[y1:y2, x1:x2] = 255

        # AQUI: enviar cb al proyector (via Syphon/NDI/ventana fullscreen)
        # ...

        # Capturar con cámara (después de tiempo de estabilización)
        cv2.waitKey(500)
        ret, captured = self.camera.read()
        gray = cv2.cvtColor(captured, cv2.COLOR_BGR2GRAY)

        # Detectar corners en la imagen capturada
        found, corners_camera = cv2.findChessboardCorners(gray, pattern_size)
        # corners_camera: coordenadas en espacio de la cámara

        # Corners conocidos en espacio del proyector
        corners_proj = np.array([
            [col*sq_size + sq_size//2, row*sq_size + sq_size//2]
            for row in range(pattern_size[1])
            for col in range(pattern_size[0])
        ], dtype=np.float32)

        if found:
            H, mask = cv2.findHomography(corners_camera, corners_proj, cv2.RANSAC)
            return H
        return None

Pipeline de Mapping 3D

import numpy as np
import open3d as o3d

class Surface3DMapper:
    """
    Video mapping sobre superficies 3D usando cámara de profundidad.
    Flujo: RGBD → Point Cloud → Mesh → UV unwrap → Warp texture
    """

    def __init__(self):
        self.pipeline = None

    def capture_rgbd(self):
        """Captura RGBD con RealSense D435 / Azure Kinect."""
        import pyrealsense2 as rs
        pipeline = rs.pipeline()
        config = rs.config()
        config.enable_stream(rs.stream.depth, 640, 480, rs.format.z16, 30)
        config.enable_stream(rs.stream.color, 640, 480, rs.format.bgr8, 30)
        pipeline.start(config)

        frames = pipeline.wait_for_frames()
        depth_frame = frames.get_depth_frame()
        color_frame = frames.get_color_frame()
        return np.asarray(depth_frame.get_data()), np.asarray(color_frame.get_data())

    def depth_to_pointcloud(self, depth: np.ndarray,
                             fx=615, fy=615, cx=320, cy=240) -> o3d.geometry.PointCloud:
        """Convierte mapa de profundidad a nube de puntos."""
        H, W = depth.shape
        u, v = np.meshgrid(np.arange(W), np.arange(H))
        Z = depth / 1000.0  # mm a metros
        X = (u - cx) * Z / fx
        Y = (v - cy) * Z / fy
        points = np.stack([X, Y, Z], axis=-1).reshape(-1, 3)
        valid = points[:, 2] > 0
        pcd = o3d.geometry.PointCloud()
        pcd.points = o3d.utility.Vector3dVector(points[valid])
        return pcd

    def project_texture_on_mesh(self, mesh: o3d.geometry.TriangleMesh,
                                  projector_matrix: np.ndarray,
                                  texture: np.ndarray) -> np.ndarray:
        """
        Proyecta textura sobre mesh 3D usando la matriz del proyector.
        Retorna frame warpado para enviar al proyector.
        """
        vertices = np.asarray(mesh.vertices)
        # Proyectar vértices al espacio del proyector
        verts_h = np.hstack([vertices, np.ones((len(vertices), 1))])
        proj_coords = (projector_matrix @ verts_h.T).T
        proj_2d = proj_coords[:, :2] / proj_coords[:, 2:3]
        return proj_2d

class GenerativeMapper:
    """Contenido procedural en tiempo real para video mapping."""

    def __init__(self, width: int = 1920, height: int = 1080):
        self.ctx = moderngl.create_standalone_context()
        self.fbo = self.ctx.framebuffer(
            color_attachments=[self.ctx.texture((width, height), 4)]
        )
        self.prog = self.ctx.program(
            vertex_shader="""
                #version 330
                in vec2 in_pos;
                out vec2 uv;
                void main() {
                    gl_Position = vec4(in_pos, 0.0, 1.0);
                    uv = in_pos * 0.5 + 0.5;
                }
            """,
            fragment_shader="""
                #version 330
                in vec2 uv;
                out vec4 fragColor;
                uniform float time;
                uniform float bass;

                void main() {
                    vec2 p = uv - 0.5;
                    float r = length(p);
                    float angle = atan(p.y, p.x);
                    float ripple = sin(r * 20.0 - time * 4.0 + bass * 15.0);
                    float brightness = (ripple * 0.5 + 0.5) * (1.0 - r * 2.0);
                    vec3 col = vec3(0.5 + 0.5*sin(angle + time),
                                   0.3 + bass * 0.7,
                                   0.8 - r);
                    fragColor = vec4(col * brightness, 1.0);
                }
            """
        )
        # Quad que cubre toda la pantalla
        vbo = self.ctx.buffer(np.array([
            -1,-1, 1,-1, -1,1, 1,1
        ], dtype='f4'))
        self.vao = self.ctx.simple_vertex_array(self.prog, vbo, 'in_pos')

    def render(self, time: float, bass: float) -> np.ndarray:
        """Renderiza frame generativo y retorna como numpy array."""
        self.fbo.use()
        self.ctx.clear(0, 0, 0)
        self.prog['time'].value = time
        self.prog['bass'].value = bass
        self.vao.render(moderngl.TRIANGLE_STRIP)
        return np.frombuffer(self.fbo.read(), dtype=np.uint8)

La Arquitectura como Personaje

En video mapping de edificios, la estructura física ES el personaje principal. El contenido debe:

  • Respetar la arquitectura: Las líneas proyectadas siguen las líneas físicas del edificio, no las cruzan arbitrariamente.
  • Revelar estructuras ocultas: El mapping puede "rayos-X" el edificio, mostrando su interior imaginario o historia.
  • Transformación progresiva: De fachada real → fachada transformada → abstracción total → retorno. Arc narrativo.
  • Climax sincronizado: El momento de mayor impacto visual coincide con el clímax musical.

Examen Feynman

Q1 · Throw Ratio

Un proyector con throw ratio 1.2 debe proyectar una imagen de 5m de ancho. ¿A qué distancia debes colocarlo? ¿Cuántos lúmenes necesitas si la superficie tiene ganancia 0.9 y quieres 20 foot-lamberts?

Ver respuesta

Distancia = R × W = 1.2 × 5m = 6m. Área = 5 × (5 × 9/16) = 5 × 2.8125 = 14.06 m² = 151.3 ft². Lúmenes = (20 ftL × 151.3 ft²) / 0.9 = 3,362 lúmenes ANSI. Un proyector de 4000 ANSI sería suficiente con margen.

Q2 · Homografía DOF

¿Por qué la homografía tiene 8 grados de libertad y no 9 (la matriz 3×3 tiene 9 elementos)?

Ver respuesta

La homografía H es una matriz proyectiva 3×3 que opera sobre coordenadas homogéneas. Cualquier escalar múltiple de H produce la misma transformación: λH·[x y 1]ᵀ y H·[x y 1]ᵀ dividen por el mismo w y dan el mismo resultado. Por convención se fija h₃₃=1 (o ||H||=1), eliminando un grado de libertad. 9 − 1 = 8 DOF: 4 puntos × 2 coordenadas = 8 ecuaciones para determinar completamente H.

Q3 · Edge Blending gamma

¿Por qué la curva de edge blending f(x)=a(2x)^p se debe aplicar ANTES de la gamma de la pantalla, y qué pasaría si se aplica después?

Ver respuesta

Las pantallas/proyectores aplican una curva gamma (≈2.2) al valor de píxel: luminancia = V^2.2. Si la máscara de blend se aplica en espacio gamma (después), la transición se verá demasiado oscura porque se comprime no-linealmente. Aplicando antes (en espacio lineal), la curva f(x) se combina linealmente con la luminancia real. El overlap de dos proyectores suma sus luminancias linealmente: L₁ + L₂ = 1.0 correctamente. Si se aplicara después en espacio gamma, la suma sería (V₁+V₂)^2.2 ≠ V₁^2.2 + V₂^2.2.

Q4 · Audio FFT

¿Qué resolución frecuencial (Hz/bin) tiene una FFT de 2048 puntos a 44100Hz? ¿Cuántos bins caen en la banda de bass [80-300Hz]?

Ver respuesta

df = fs/N = 44100/2048 = 21.5 Hz/bin. Bin mínimo de bass: floor(80/21.5) = bin 3. Bin máximo de bass: floor(300/21.5) = bin 13. Número de bins = 13 − 3 = 10 bins en la banda [80-300Hz]. Para mejor resolución en bajas frecuencias se puede usar FFT más larga (4096, 8192) o análisis multi-resolución (CQT).

Q5 · DLT mínimo

¿Por qué se necesitan mínimo 4 puntos para calcular una homografía con DLT? ¿Qué pasa si los 4 puntos son colineales?

Ver respuesta

Cada correspondencia de puntos aporta 2 ecuaciones al sistema Ah=0. Una homografía tiene 8 DOF (incógnitas), por lo que necesitamos 8/2 = 4 puntos mínimo para un sistema determinado. Si los 4 puntos son colineales (todos en la misma recta), el sistema es degenerado: la solución SVD no determina H de forma única. Existen múltiples homografías que mapean esos 4 puntos colineales correctamente (la geometría 2D se colapsa en 1D). Por eso RANSAC y los sistemas de mapping exigen que los puntos de control cubran el área 2D y no sean colineales.
